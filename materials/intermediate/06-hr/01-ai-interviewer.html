AI面接官：リアルタイム質問・回答収集・採点

★★★★ エンジニア向け 構築数日〜 Claude Code 音声認識 採点ロジック
← 一覧に戻る

概要

オンライン面接（Zoom/Google Meet等）でAIが面接官として質問を行い、応募者の回答をリアルタイムでテキスト化・採点・記録するシステム。一次面接のスクリーニングを自動化し、採用担当者の工数を削減する。

■ 懸念点
- コミュニケーションの平易化トレーニングが有効かもしれない
- チームマネジメント経験はゼロ（リーダー職には追加確認が必要）

■ 推奨質問（次の面接担当者向け）
1. 非技術者のステークホルダーと連携した経験について
2. リモートチームでの協働経験について

重要な留意事項

AIによる採用選考は公正採用の観点から慎重な運用が必要です。AIの評価は一次スクリーニングの参考情報として使用し、採用可否の最終決定は必ず人間が行ってください。また、応募者に対してAI面接であることを事前に告知することが求められます。

システム構成（テキストベース版）

# 音声なしのシンプルな構成（推奨スタート地点） 1. 応募者がWebフォームでURLにアクセス 2. AIが質問を1問ずつテキスト表示 3. 応募者がテキストで回答を入力 4. AIが次の質問を生成（回答に応じた深掘り質問も可能） 5. 面接終了後に評価レポートを採用担当者に送信

設定手順

  • 1
    評価基準と質問バンクを設計する
    # evaluation_criteria.md（例） ## 評価軸 1. 論理的思考力（5点満点）：理由を順序立てて説明できるか 2. 問題解決力（5点満点）：困難な状況への対処法を具体的に語れるか 3. コミュニケーション力（5点満点）：簡潔かつ明確に伝えられるか 4. カルチャーフィット（5点満点）：自社の価値観との合致 ## 質問バンク（抜粋） - 自己紹介（3分以内） - これまでのキャリアで最も困難だった経験と乗り越え方 - なぜ当社に興味を持ったのか - 5年後のキャリアビジョン
  • 2
    面接ウェブアプリをClaude Codeで構築

    シンプルなHTMLフォームとPython（Flask/FastAPI）バックエンドで構成。セッション管理で面接の進行状態を保持する。

  • 3
    採点ロジックを実装する
    # scorer.py（抜粋） def score_response(question: str, answer: str, criteria: str) -> dict: client = anthropic.Anthropic() response = client.messages.create( model="claude-opus-4-7", messages=[{"role": "user", "content": f"面接の質問と回答を評価基準に沿って採点してください。\n" f"質問：{question}\n回答：{answer}\n" f"評価基準：{criteria}\n\n" f"各評価軸のスコア（5点満点）と根拠を JSON で返してください。"}] ) return parse_scores(response.content[0].text)
  • 4
    評価レポートを採用担当者に送信

    面接終了後に採点結果・回答の要約・合否推奨をメールで自動送信する。採用担当者はレポートを元に最終判断を行う。

設定例（ケース別）

技術職採用 — 評価軸・質問バンク設定例
【評価軸（5点満点）】
1. 技術的専門性：該当技術スタックへの習熟度
2. 問題解決力：デバッグ・設計判断のプロセス
3. コード品質への意識：テスト・レビュー・ドキュメント
4. チームワーク：コードレビュー・ペアプロ経験

【質問バンク（抜粋）】
- 得意な技術スタックとその選定理由を教えてください
- 本番障害の対応経験と、その時の判断プロセスを教えてください
- コードレビューで気をつけていることは何ですか？
- 技術的な負債に直面したとき、どのようにアプローチしますか？
営業職採用 — 評価軸・質問バンク設定例
【評価軸（5点満点）】
1. 傾聴力：顧客ニーズの引き出し方
2. 提案力：課題解決型の提案ができるか
3. 粘り強さ：断られた経験と乗り越え方
4. 数字への意識：目標・実績の把握と管理

【質問バンク（抜粋）】
- これまでの営業成績で最も誇れる実績を教えてください
- 断られた案件を覆した経験はありますか？
- 顧客のニーズを引き出すために意識していることは？
- ノルマ未達成だったとき、どのように対処しましたか？
管理職採用 — 評価軸・質問バンク設定例
【評価軸（5点満点）】
1. リーダーシップ：チームを動機づける方法
2. 意思決定力：不確実な状況での判断
3. 人材育成：メンバーの成長へのアプローチ
4. 戦略的思考：中長期の視点を持った計画立案

【質問バンク（抜粋）】
- チームが機能不全に陥ったとき、どのように立て直しましたか？
- メンバーのパフォーマンスが低下したとき、どう対処しますか？
- 自部門の中長期目標をどのように設定・管理しますか？
- 経営層と現場の板挟みになった経験はありますか？

確認チェックリスト

  • 応募者にAI面接であることを事前に告知・同意を取得しているか
  • 評価軸が応募職種に直接関係する能力のみに絞られているか（性別・年齢・出身地等は含まれていないか）
  • AIの採点結果はあくまで参考情報として扱い、採用可否の最終決定を人間が行う運用になっているか
  • AIの採点に性別・年齢・出身等のバイアスが生じていないか定期的に監査しているか
  • 応募者の回答データがHTTPS・DB暗号化・アクセス制限で適切に保護されているか
  • 外部AIサービスへのデータ送信について個人情報取り扱い規約を確認しているか
  • 採点レポートにアクセスできる社内担当者を最小限に絞っているか

注意点

  • 応募者の回答テキストは個人情報。適切なセキュリティ措置（HTTPS・DB暗号化・アクセス制限）を施す。
  • AIの採点結果に偏り（性別・年齢・出身等に関する偏見）が生じていないか定期的に監査する。
  • AIはバイアスがかかる可能性があり、採用可否の最終決定は必ず人間が行う。AI評価はあくまで一次スクリーニングの参考情報として扱うこと。
  • 外部サービス（OpenAI等）にデータを送る場合は個人情報の取り扱い規約を確認し、応募者の同意を得ること。
