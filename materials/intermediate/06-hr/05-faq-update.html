社内FAQ・ナレッジベースの自動更新

★★★ 設定半日〜 FAQを常に最新に保つ Claude Code Slack MCP Notion/GitBook
← 一覧に戻る

概要

Slackの質問チャンネルを監視し、繰り返し質問されている内容をClaudeが検出・整理してFAQに追加するフローを構築する。ナレッジベースが常に最新の状態に保たれ、同じ質問への対応時間を削減できる。

設定手順

  • 1
    質問収集の仕組みを設計する

    情報源として以下を使用する：
    ・Slackの #質問 #help チャンネルのログ
    ・サポートメールのスレッド
    ・社内問い合わせフォームの履歴

  • 2
    月次FAQ更新スクリプトを作成（Claude Codeで生成）
    # faq_updater.py（Claude Codeで生成・抜粋） def extract_new_faq_candidates(slack_messages: list, existing_faq: str) -> list: client = anthropic.Anthropic() messages_text = "\n".join([f"Q: {m['text']}" for m in slack_messages]) response = client.messages.create( model="claude-opus-4-7", messages=[{"role": "user", "content": f"以下のSlackメッセージを分析し、FAQに追加すべき質問を抽出してください。\n" f"既存のFAQ：\n{existing_faq}\n\n" f"新規質問一覧：\n{messages_text}\n\n" f"既存FAQと重複しない、かつ3回以上登場する質問のみを抽出し、\n" f"Q/A形式で回答案も生成してください。"}] ) return parse_faq_candidates(response.content[0].text)
  • 3
    レビューワークフローを設定する

    自動生成されたFAQ候補を担当者がSlackで確認・承認できるフローを設定。承認されたFAQのみをNotionに自動追加する。

    # Slackに承認リクエストを送信 新しいFAQ候補が〇件見つかりました。 以下のリンクから確認・承認してください： [FAQ承認ページ]（https://...）
  • 4
    古くなったFAQの検出と削除提案

    過去6ヶ月参照されていないFAQ・製品バージョンが古いFAQをClaudeが検出して担当者に削除・更新を提案する機能を追加する。

注意点

  • AI生成のFAQ回答は必ず担当者がレビューしてから公開する。誤った回答がナレッジベースに混入すると、誤情報が拡散するリスクがある。
  • 個人情報が含まれるSlackメッセージをAIに渡す場合は、個人名・プロジェクト名等を匿名化してから処理する。
  • FAQの更新頻度は月1回程度が目安。更新しすぎると現場の混乱を招く。
