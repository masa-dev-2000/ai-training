「議事録まとめて」スラッシュコマンド化・1発実行

★★ 設定1〜2時間 テキストを貼って1コマンド Claude Code Skills
← 一覧に戻る

概要

会議中にとったメモ・チャットログ・文字起こしテキストをClaude Codeに貼り付けて /minutes と入力するだけで、決定事項・アクションアイテム・次回会議日程が整理された議事録を1発で生成するスキルコマンドを設定する。

設定手順

  • 1
    議事録テンプレートを決める

    チーム・会議種別によってテンプレートを用意する。「週次MTG」「プロジェクト会議」「1on1」等で別テンプレートを作ると用途に合った議事録が生成される。

  • 2
    スキルファイルを作成する
    # .claude/skills/minutes.md --- name: minutes description: 会議メモから議事録を生成する --- 以下の手順で議事録を生成してください： 1. ユーザーから会議の内容（メモ・チャット・文字起こし）を受け取る 2. 以下のフォーマットで議事録をMarkdownで出力する： --- # 会議名（メモから推定） 日時：〇〇年〇月〇日 参加者：〇〇（メモから抽出） ## 議題 1. （一覧） ## 決定事項 - 〇〇を△△に変更することになった - □□プロジェクトは来月末を目標に進める ## アクションアイテム | 担当者 | タスク | 期限 | |--------|--------|------| | 〇〇さん | △△の調査 | 〇月〇日 | ## 次回会議 日時・場所（言及があれば） ## 備考・懸案事項 --- 3. 生成後「Notionにも保存しますか？」と確認する
  • 3
    実行してみる
    使い方
    /minutes

（以下に会議メモをペースト）
参加：田中、山田、佐藤
・新機能リリースは来月15日に決定
・テスト担当は山田さん、来週金曜までに完了
・競合調査を佐藤さんが2週間以内に実施
・次回MTGは来週水曜10時〜
  • 4
    Notion連携を追加する（任意）

    Notion MCPをClaude Codeに追加すると、議事録生成後に「Notionに保存しますか？」の確認を経てNotionデータベースに自動保存できる。

    # .claude/settings.jsonにNotion MCPを追加 { "mcpServers": { "notion": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-notion"], "env": { "NOTION_API_KEY": "secret_xxx" } } } }

注意点

  • アクションアイテムの担当者名と期限は、メモに明記されていない場合AIが推測することがある。送信前に必ず確認する。
  • 会議参加者の個人名が含まれる場合、Claudeのプロジェクト設定でデータ保持をオフにして使う。
  • 決定事項の言い回しはAIが整形するため、原文のニュアンスが変わることがある。重要な決定は原文と照合する。
