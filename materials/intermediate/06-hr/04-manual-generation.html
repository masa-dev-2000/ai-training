操作録画 ＋ ナレーション → Markdownマニュアル自動生成

★★★★ エンジニア向け マニュアル作成を全自動化 FFmpeg Whisper Claude Code
← 一覧に戻る

概要

操作手順を録画したMP4ファイル（ナレーション付き）を入力として、スクリーンショット抽出・ナレーション文字起こし・操作手順の構造化を自動で行い、スクリーンショット付きMarkdownマニュアルを出力する。

生成された Markdown マニュアル（manual.md 冒頭）
# 〇〇システム 操作マニュアル
> 最終更新：2026-05-15 | 対象バージョン：v2.4.0 | 対象者：一般ユーザー

## はじめに
このマニュアルでは〇〇システムへのログインから基本操作までを説明します。
不明点は社内ヘルプデスク（内線 1234）までお問い合わせください。

## 手順1：ログイン

![ログイン画面](screenshots/frame_0003.png)

1. ブラウザ（Chrome推奨）で `https://system.example.co.jp` にアクセスする
2. **メールアドレス**欄に社内メールアドレスを入力する
3. **パスワード**欄にパスワードを入力する（初回ログイン時は仮パスワードを使用）
4. 「ログイン」ボタンをクリックする

> ⚠️ パスワードを3回間違えるとアカウントがロックされます。
> ロック解除は管理者（admin@example.co.jp）に依頼してください。

## 手順3：新規案件の登録

![新規案件登録画面](screenshots/frame_0021.png)

1. 左メニューの「案件管理」をクリックする
2. 右上の「＋ 新規登録」ボタンをクリックする
3. 必須項目（赤枠）をすべて入力する
4. 「保存」ボタンをクリックする → 登録完了メッセージが表示される

（以下、手順4〜12 が続く）

処理パイプライン

# 処理の流れ 録画MP4 ↓ FFmpegでスクリーンショット抽出（1秒ごと） ↓ Whisperでナレーション文字起こし ↓ タイムスタンプを使って「画面変化点」を検出 ↓ 画面変化点のスクリーンショット + 対応するナレーションを紐付け ↓ Claudeで操作手順を構造化 ↓ Markdownマニュアル出力（スクリーンショット埋め込み）

必要な準備

FFmpeg（インストール必要） OpenAI Whisper（pip install openai-whisper） Pillow（スクリーンショット処理） Claude Code

設定手順

  • 1
    操作録画を撮影する

    OBS Studio・QuickTime・Windows標準録画等で操作手順を録画。ナレーションを入れながら録画すると精度が上がる。ナレーションがない場合はテロップを付けた動画でも対応可能。

  • 2
    スクリーンショット抽出
    # 1秒ごとにスクリーンショット抽出 ffmpeg -i input.mp4 -vf fps=1 screenshots/frame_%04d.png
  • 3
    ナレーション文字起こし
    # Whisperで文字起こし（タイムスタンプ付き） whisper input.mp4 --language ja --output_format json --output_dir ./
  • 4
    マニュアル生成スクリプトをClaude Codeで作成

    タイムスタンプを使ってスクリーンショットとナレーション文字起こしを紐付け、Claudeで操作手順を構造化するスクリプトを生成してもらう。

    # 出力例（manual.md） # 〇〇の操作マニュアル ## 手順1：ログイン ![ログイン画面](screenshots/frame_0001.png) 1. ブラウザで https://... にアクセスする 2. メールアドレスとパスワードを入力する 3. 「ログイン」ボタンをクリックする

設定例（ケース別）

操作手順書 — 構造化プロンプト設定例
以下のナレーション文字起こしとスクリーンショット情報から
操作手順書を生成してください。

【出力形式】
- タイトル・対象バージョン・対象者を冒頭に記載
- 各手順は「## 手順N：（操作名）」で始める
- スクリーンショットを手順ごとに `![説明](path)` で挿入
- 番号付きリストで操作を1ステップ1行で記載
- 注意・警告事項は `> ⚠️` で記載

【対象者】一般業務ユーザー（IT知識は不問）
【口調】丁寧だが簡潔。専門用語は使わない
トラブルシューティングガイド — 構造化プロンプト設定例
以下のナレーション・画面情報からトラブルシューティングガイドを生成してください。

【出力形式】
各トラブルは以下の構造で記載する：
---
### ❌ 症状：（画面に何が起きているか）
**原因：**（なぜこの問題が起きるか）
**解決手順：**
1. （手順1）
2. （手順2）
**解決しない場合：** （次の連絡先・エスカレーション先）
---

スクリーンショットはエラー画面のみ挿入する
「よくある質問（FAQ）」セクションを末尾に追加する
新人向けオンボーディング — 構造化プロンプト設定例
以下の操作録画から新入社員向けのオンボーディングガイドを生成してください。

【出力形式】
- 冒頭に「このガイドを読み終えたら何ができるか」を3点で明示
- 操作手順は丁寧に（なぜその操作をするかの理由も添える）
- 「よくある間違い」コラムを各章末尾に追加
- 「困ったときの連絡先」セクションを末尾に記載

【対象者】PC操作は慣れているが業務システムは初めての新入社員
【口調】友好的で励ます表現。「まず〇〇してみましょう」など促す語調

確認チェックリスト

  • スクリーンショットに個人情報・パスワード・機密データが映り込んでいないか全件確認しているか
  • 手順番号とスクリーンショットが正しく対応しているか（ズレていないか）
  • ナレーションの誤認識による手順の誤りがないか確認しているか
  • 専門用語・略語に説明が添えられているか（対象読者に合わせて）
  • マニュアルの公開先（社内Wiki・Notion等）のアクセス権限が適切に設定されているか
  • システムのバージョン情報・作成日・作成者がマニュアルに明記されているか

注意点

  • 画面変化が少ない操作（文字入力・スクロール等）は変化点検出が難しい。ナレーションの区切りでステップを区切る実装が安定する。
  • スクリーンショットに個人情報・パスワードが映り込まないよう注意。マニュアル公開前に全スクショを確認する。
  • Whisperのローカル実行はGPUがあると高速。CPU環境では長い動画の文字起こしに時間がかかる。
