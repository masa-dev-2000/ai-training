Zoom録画 → 構造化議事録 → Slack/Notion自動投稿

★★★ 設定半日〜 会議後の議事録作成を自動化 Zoom API Whisper / AssemblyAI Notion MCP
← 一覧に戻る

概要

Zoomのクラウド録画が完了すると自動でZoom APIから音声ファイルを取得、文字起こしを行い、Claudeが構造化議事録を生成してSlackとNotionに自動投稿するパイプラインを構築する。

必要な準備

Zoom Pro以上のアカウント（クラウド録画有効） Zoom OAuth App（recordings:read スコープ） AssemblyAI または OpenAI Whisper APIキー Slack Webhook / Notion MCP

設定手順

  • 1
    Zoom Webhookを設定して録画完了イベントを受信

    Zoom Apps Marketplace → Webhook Only App作成 → recording.completed イベントを購読。録画完了時に自動でパイプラインが起動する。

  • 2
    音声ファイルを取得して文字起こし
    # transcriber.py（抜粋） import assemblyai as aai def transcribe_audio(audio_url: str) -> str: aai.settings.api_key = ASSEMBLYAI_API_KEY config = aai.TranscriptionConfig( speaker_labels=True, # 話者識別 language_code="ja" ) transcript = aai.Transcriber().transcribe(audio_url, config) # 話者ラベル付きテキストを整形 return format_transcript(transcript.utterances)

    Slack/Notionに自動投稿

    生成した議事録をSlack Incoming Webhookで該当チャンネルに投稿し、Notion MCPで議事録データベースに追加する。

経営会議（機密レベル高）— 設定例
【機密対応設定】
- 文字起こしデータは暗号化ストレージにのみ保存
- 外部AIサービス（AssemblyAI等）への送信は禁止
  → Whisperをオンプレミス・ローカル実行に切り替え
- 議事録の投稿先はSlack ではなく 社内の権限付きNotionページのみ
- 閲覧権限は参加者と指定役員のみ

【出力形式】
- 参加者（役職付き）
- 経営上の決定事項（箇条書き）
- 対外秘項目の明示（⚠️ 機密マーク付き）
- アクションアイテム（担当役員・期限）
- 次回経営会議日時

【注意】議事録の承認は社長が行ってから配布する

確認チェックリスト

  • 会議開始時に参加者全員に録画・AI文字起こしの実施を告知しているか
  • アクションアイテムの担当者名と期限が正確に記録されているか（AIが推測したものは要確認）
  • 固有名詞（社名・製品名・人名）の誤変換がないか目視確認しているか
  • 機密情報を含む会議の議事録を外部サービスに送信していないか
  • 議事録の共有範囲が会議の機密レベルに合っているか（全社共有か、参加者のみか）
  • Notionへの投稿が正常に完了しているか（リンク切れ・権限エラーがないか）

注意点

  • 会議の録画・文字起こしには参加者の同意が必要。会議開始時に「録画・AI文字起こしを行う」旨を案内する。
  • AssemblyAIの日本語対応は良好だが、専門用語・固有名詞の精度は低い。固有名詞辞書の設定（Custom Vocabulary）で改善できる。
  • Zoomのクラウド録画は長時間・高頻度の会議が多い場合、ストレージ費用がかかる。ローカル録画との使い分けを検討する。
