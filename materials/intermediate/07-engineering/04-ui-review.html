概要

/ui-review [URL] を実行すると、Playwrightが指定URLのブラウザ操作を自動実行・スクリーンショットを撮影し、Claudeがデザイン品質・UX問題・アクセシビリティ・モバイル対応をAI分析したレポートを生成する。

【優先度：高】改善提案
1. モバイル画面でログインボタンがビューポートの下に隠れている
   → ボタンを画面上部に移動するか、スクロール不要なレイアウトに変更を推奨
2. パスワード入力欄にラベルがなく、プレースホルダーのみ
   → スクリーンリーダーがラベルを読み上げられない。<label>要素を追加すること

【優先度：中】改善提案
3. フォームの背景色（#F5F5F5）とテキスト色（#767676）のコントラスト比が3.2:1
   → WCAG AA基準（4.5:1）を満たしていない。テキスト色を #595959 以上の濃さに変更推奨
4. ソーシャルログインボタン（Google・GitHub）とメールログインフォームの区別が不明確
   → 「または」などの区切りテキストと水平線で視覚的に分離することを推奨

【優先度：低】改善提案
5. ページタイトルが「ログイン」のみで会社名が含まれていない
   → 「ログイン | サービス名」の形式に変更推奨（SEO・ブックマーク識別のため）

設定例（ケース別）

デスクトップ確認 — スキル設定例（ui-review.md）
---
name: ui-review
description: URLのUIをPlaywrightでスクショしてAI分析レポートを生成する
---

1. 引数のURLをui_screenshotter.jsに渡す
   - デスクトップ: 1280×800, 1920×1080 の2サイズ
2. analyze_ui.py で以下を分析：
   - 視覚的な一貫性（カラーパレット・フォント・余白）
   - 情報設計（コンテンツの優先度・視線の流れ）
   - CTAボタンの明確さ・視認性
   - フォーム設計（ラベル・エラー表示・入力ガイド）
3. レポートを reports/ui-review-[ページ名]-[日付].md に保存
モバイル確認 — スキル設定例（ui-review-mobile.md）
---
name: ui-review-mobile
description: モバイル表示に特化したUIレビューを実施する
---

1. 引数のURLを以下の3デバイスサイズでスクショ撮影：
   - iPhone 15 Pro: 393×852
   - Galaxy S24: 360×800
   - iPad: 820×1180
2. モバイル特有の観点で分析：
   - タップターゲットのサイズ（最低44×44px）
   - ビューポート内に主要なCTAが収まっているか
   - 横スクロールが発生していないか
   - フォント・ボタンの文字が読みやすいか（最低14px相当）
   - ナビゲーションがモバイルに最適化されているか（ハンバーガーメニュー等）
3. レポートを reports/ui-review-mobile-[日付].md に保存
アクセシビリティ確認 — スキル設定例（ui-review-a11y.md）
---
name: ui-review-a11y
description: アクセシビリティ観点に特化したUIレビューを実施する
---

1. 引数のURLをPlaywrightで開き、以下を実行：
   - スクリーンショット撮影（通常表示）
   - axe-core ライブラリでアクセシビリティ自動検査を実行
2. Claude で以下の観点で分析：
   - カラーコントラスト比（WCAG AA基準: 4.5:1以上）
   - 画像のalt属性の適切な記述
   - フォームの<label>要素の存在
   - キーボードナビゲーションの可否（視覚的フォーカスインジケータ）
   - ARIAランドマーク（header/main/footer）の設定
3. WCAG 2.1 AA 基準に対する適合状況をスコア化してレポート出力

設定手順

  • 1
    Playwrightをインストール
    npm install playwright npx playwright install chromium
  • 2
    スクリーンショット撮影スクリプトをClaude Codeで生成
    # ui_screenshotter.js（Claude Codeで生成） const { chromium } = require('playwright'); async function captureScreenshots(url, outputDir) { const browser = await chromium.launch(); const context = await browser.newContext(); // デスクトップ const desktopPage = await context.newPage(); await desktopPage.setViewportSize({ width: 1280, height: 800 }); await desktopPage.goto(url, { waitUntil: 'networkidle' }); await desktopPage.screenshot({ path: `${outputDir}/desktop.png`, fullPage: true }); // モバイル const mobilePage = await context.newPage(); await mobilePage.setViewportSize({ width: 390, height: 844 }); await mobilePage.goto(url); await mobilePage.screenshot({ path: `${outputDir}/mobile.png`, fullPage: true }); await browser.close(); }
  • 3
    AI分析レポート生成スクリプトを追加
    # analyze_ui.py（Claude Codeで生成・抜粋） import anthropic, base64 def analyze_screenshots(desktop_path: str, mobile_path: str, url: str) -> str: with open(desktop_path, "rb") as f: desktop_b64 = base64.b64encode(f.read()).decode() with open(mobile_path, "rb") as f: mobile_b64 = base64.b64encode(f.read()).decode() client = anthropic.Anthropic() response = client.messages.create( model="claude-opus-4-7", max_tokens=2000, messages=[{"role": "user", "content": [ {"type": "text", "text": f"以下のWebページ（{url}）のUI/UXレビューを実施してください。"}, {"type": "image", "source": {"type": "base64", "media_type": "image/png", "data": desktop_b64}}, {"type": "image", "source": {"type": "base64", "media_type": "image/png", "data": mobile_b64}}, {"type": "text", "text": """ 評価観点： 1. 視覚的な一貫性とデザイン品質（スコア/5点） 2. 情報設計・ナビゲーション（スコア/5点） 3. モバイル対応（スコア/5点） 4. アクセシビリティ（推定スコア/5点） 5. CTAの明確さ（スコア/5点） 優先度別の改善提案（高/中/低）を箇条書きで出力してください。 """} ]}] ) return response.content[0].text
  • 4
    スキルコマンドとして登録
    # .claude/skills/ui-review.md --- name: ui-review description: URLのUIをPlaywrightでスクショしてAI分析レポートを生成する --- 1. 引数のURLをui_screenshotter.jsに渡してスクショ撮影 2. analyze_ui.pyで分析レポートを生成 3. レポートをMarkdownファイルに保存

確認チェックリスト

  • スクリーンショット撮影対象のURLに誤りがないか（本番URLを誤って含めていないか）
  • ログインが必要なページは認証情報をPlaywrightに正しく渡せているか
  • 分析レポートのスコア・指摘事項をデザイナー・フロントエンド担当が確認したか
  • 「優先度：高」の指摘に対してチケット（Issue・Jiraタスク）を起票したか
  • Claude APIのビジョン機能の利用コストを確認し、分析ページ数が適切か
  • アクセシビリティ指摘については自動検査（axe-core等）と目視確認の両方で検証したか
  • AIのUI評価はあくまで参考情報として扱い、最終判断はデザイナーが行ったか

注意点

  • ログインが必要なページは認証情報をPlaywrightに渡す処理が別途必要。
  • Claude APIのビジョン機能（画像解析）はコストが高め。大量のページを一度に分析すると費用がかさむ。
  • AIのUI評価は主観的な要素を含む。あくまで改善のヒントとして使い、最終判断はデザイナー・担当者が行う。
