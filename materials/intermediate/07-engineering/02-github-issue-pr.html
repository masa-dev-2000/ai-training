GitHub Issueに @claude でメンション → 実装 → PR自動作成

★★★★ エンジニア向け Issue起票からPRまで自動 GitHub MCP Claude Code GitHub Actions
← 一覧に戻る

概要

GitHubのIssueコメントで @claude implement this とメンションすると、Claude CodeがIssueの内容を読み取り・コードを実装・テストを書き・PRを自動作成するワークフローを構築する。小規模なバグ修正・機能追加を高速化できる。

完成イメージ（出力例）

↓ Issue #42「ログインページのバリデーションエラー表示が不正」に @claude implement this とコメントした場合の自動作成PRの例

Claudeが自動作成したPRの概要
PR #44: fix: ログインページのバリデーションエラー表示を修正
ブランチ: claude/fix-login-validation-42
作成者: @claude-bot

---

Issue #42 の問題を修正しました。メールアドレス形式が不正な場合にエラーメッセージが

設定例（ケース別）

バグ修正Issue — Claudeへの指示テンプレート
以下のバグ修正Issueを実装してPRを作成してください。

Issue内容：$ISSUE_BODY

実装方針：
- 最小限の変更でバグを修正する（スコープを広げない）
- 修正した箇所の単体テストを追加する
- 修正内容をPR本文に「再現手順」「原因」「修正内容」の形式でまとめる

完了後：
- git add → git commit（"fix: [Issue内容を要約]"）
- feature/fix-[issue番号] ブランチを作成してPRを作成
機能追加Issue — Claudeへの指示テンプレート
以下の機能追加Issueを実装してPRを作成してください。

Issue内容：$ISSUE_BODY

実装方針：
- CONTRIBUTING.md のコーディング規約に従う
- 新機能に対応するユニットテストを追加する（カバレッジ80%以上）
- 変更が大きい場合は実装前に設計をコメントとして確認を取る
- APIの変更がある場合はOpenAPI仕様書（docs/api.yaml）も更新する

完了後：
- git add → git commit（"feat: [機能名]"）
- feature/[機能名]-[issue番号] ブランチを作成してPRを作成
- PRのラベルに "enhancement" を付ける
リファクタリングIssue — Claudeへの指示テンプレート
以下のリファクタリングIssueを実施してPRを作成してください。

Issue内容：$ISSUE_BODY

実装方針：
- 外部から見える振る舞いを変えない（既存テストがすべてパスすること）
- リファクタリング前後で動作が同一であることをテストで確認する
- 変更の意図（なぜ変えるか）をコードコメントまたはPR本文に記載する
- 1つのPRで変更するファイル数は10ファイル以内に収める

完了後：
- git add → git commit（"refactor: [変更内容を要約]"）
- refactor/[内容]-[issue番号] ブランチを作成してPRを作成

実装方法（2通り）

# 方法A: GitHub ActionsワークフローでClaude Codeを起動 Issueコメント（@claude） → GitHub Actionsトリガー → Claude Code実行（ヘッドレスモード） → コード生成・コミット・PR作成 # 方法B: Claude Code for GitHub（公式機能） GitHubアプリ「Claude」をリポジトリにインストール → Issueで@claudeメンション → Claudeが自動実装・PR作成

方法A：GitHub Actionsで自動化する手順

  • 1
    GitHub Actionsワークフローを作成
    # .github/workflows/claude-implement.yml name: Claude Auto-Implement on: issue_comment: types: [created] jobs: implement: if: contains(github.event.comment.body, '@claude implement') runs-on: ubuntu-latest permissions: contents: write pull-requests: write issues: write steps: - uses: actions/checkout@v4 - name: Install Claude Code run: npm install -g @anthropic-ai/claude-code - name: Run Claude Code env: ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }} GH_TOKEN: ${{ secrets.GITHUB_TOKEN }} run: | ISSUE_BODY="${{ github.event.issue.body }}" claude --headless --print " 以下のIssueを実装してPRを作成してください： $ISSUE_BODY 実装後、git add → commit → branchを作成してGitHub MCPでPRを作成すること。 "
  • 2
    ANTHROPIC_API_KEYをSecretsに登録

    リポジトリ設定 → Secrets and variables → Actions → ANTHROPIC_API_KEY を追加する。

  • 3
    実装の品質を確保するためのガードレールを設定

    Claudeに渡す指示に「テストを必ず書く」「既存のコーディング規約（CONTRIBUTING.md参照）に従う」「大きな変更の場合は実装前に確認コメントを入れる」を追加する。

確認チェックリスト

  • 自動作成されたPRのコードを人間がレビューしたか（自動マージ設定になっていないか）
  • CIのすべてのテスト・リントチェックがパスしているか
  • Claudeがトリガーできるユーザーをホワイトリストで制限しているか
  • ANTHROPIC_API_KEY が GitHub Secrets に安全に保存されているか
  • 本番環境への意図しないデプロイが走らないよう、ブランチ保護ルールを設定しているか
  • 月次のAPI利用コストをモニタリングし、上限を設定しているか
  • 大きな機能変更の場合、実装前にClaudeが設計確認コメントを投稿する設定になっているか

注意点

  • 誰でも @claude メンションで自動実装をトリガーできると危険。コントリビューターのホワイトリストを設けるか、mainブランチへの直接pushを禁止する。
  • 生成されたコードは必ずコードレビューしてからマージする。自動実装のPRをレビューなしで自動マージする設定は避ける。
  • Claude Code APIの利用コストが発生する。月次の利用量をモニタリングしてコスト上限を設定しておく。
  • 既存テストが通ることをCIで確認してからマージすること。
