ドキュメントのスクリーンショット更新を全自動化

★★★ 設定半日〜 UI変更時に自動でドキュメント更新 Playwright GitHub Actions Claude Code
← 一覧に戻る

概要

UIの変更がmainブランチにマージされるとGitHub Actionsが自動起動し、Playwrightがスクリーンショットを再撮影。既存のドキュメント画像と差分を検出し、変更があれば自動でドキュメントを更新してPRを作成する。

設定例（ケース別）

単一ページ更新 — GitHub Actions 設定例
# .github/workflows/update-single-screenshot.yml
name: Update Single Page Screenshot
on:
  workflow_dispatch:
    inputs:
      page_id:
        description: '更新するページID（screenshots.config.json のid）'
        required: true
        type: string

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci && npx playwright install chromium
      - run: node scripts/update-screenshots.js --page ${{ inputs.page_id }}
      - name: Create PR if changed
        uses: peter-evans/create-pull-request@v5
        with:
          commit-message: "docs: ${{ inputs.page_id }} のスクリーンショットを更新"
          title: "[${{ inputs.page_id }}] ドキュメント画像を手動更新"
          branch: "manual/update-screenshot-${{ inputs.page_id }}"
全ページ一括更新 — 週次スケジュール実行設定例
# .github/workflows/weekly-screenshot-update.yml
name: Weekly Screenshot Update
on:
  schedule:
    - cron: '0 2 * * 1'  # 毎週月曜 2:00 UTC（日本時間 11:00）
  workflow_dispatch:

jobs:
  update-all:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci && npx playwright install chromium
      - run: node scripts/update-screenshots.js --all
      - name: Create PR if any changed
        uses: peter-evans/create-pull-request@v5
        with:
          commit-message: "docs: スクリーンショットを週次一括更新"
          title: "ドキュメントのスクリーンショット週次自動更新"
          branch: "auto/weekly-screenshots"
          labels: "documentation,automated"
認証が必要なページ — Playwright 設定例
// scripts/update-screenshots.js（認証あり部分）
async function loginAndCapture(page, item) {
  // ログイン処理
  await page.goto(process.env.BASE_URL + '/login');
  await page.fill('[name="email"]', process.env.TEST_USER_EMAIL);
  await page.fill('[name="password"]', process.env.TEST_USER_PASSWORD);
  await page.click('[type="submit"]');
  await page.waitForURL('**/dashboard');

  // 認証済みページのスクリーンショット撮影
  await page.goto(process.env.BASE_URL + item.url);
  return await page.screenshot({ fullPage: true });
}

// GitHub Secretsに設定する環境変数
// TEST_USER_EMAIL: テスト用アカウントのメールアドレス
// TEST_USER_PASSWORD: テスト用アカウントのパスワード
// BASE_URL: 対象環境のベースURL

設定手順

  • 1
    スクリーンショット設定ファイルを作成
    # docs/screenshots.config.json { "screenshots": [ { "id": "login-page", "url": "/login", "viewport": {"width": 1280, "height": 800}, "output": "docs/images/login-page.png", "steps": [ {"action": "goto", "url": "/login"} ] }, { "id": "dashboard", "url": "/dashboard", "viewport": {"width": 1280, "height": 800}, "output": "docs/images/dashboard.png", "auth": true, "steps": [ {"action": "login"}, {"action": "goto", "url": "/dashboard"} ] } ] }
  • 2
    スクリーンショット撮影スクリプトを作成
    # scripts/update-screenshots.js（Claude Codeで生成） const { chromium } = require('playwright'); const config = require('../docs/screenshots.config.json'); async function updateScreenshots() { const browser = await chromium.launch(); const changed = []; for (const item of config.screenshots) { const page = await browser.newPage(); await page.setViewportSize(item.viewport); for (const step of item.steps) { await executeStep(page, step); } const newShot = await page.screenshot(); if (await hasChanged(item.output, newShot)) { await fs.writeFile(item.output, newShot); changed.push(item.id); } } await browser.close(); return changed; }
  • 3
    GitHub Actionsワークフローを設定
    # .github/workflows/update-docs-screenshots.yml name: Update Documentation Screenshots on: push: branches: [main] paths: ['src/**', 'public/**'] jobs: update-screenshots: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - run: npm ci && npx playwright install - run: node scripts/update-screenshots.js - name: Create PR if changed uses: peter-evans/create-pull-request@v5 with: commit-message: "docs: スクリーンショットを最新UIに更新" title: "ドキュメントのスクリーンショット自動更新" branch: "auto/update-screenshots"

確認チェックリスト

  • 自動作成されたPRの画像を目視で確認し、意図しない変更が含まれていないか
  • ドキュメントの文章と更新後の画像の内容が一致しているか（手順番号・ボタン名等）
  • テスト用アカウントの認証情報をGitHub Secretsに安全に保存しているか
  • 差分検出の閾値（ピクセル差分 %）を適切に設定しているか（小さすぎると誤検出、大きすぎると見落とし）
  • CI実行時間がプロジェクトの許容範囲内か（長ければ対象ページを絞るか週次実行に変更）
  • 自動更新PRをマージする前にレビュアーが承認する設定になっているか
  • 本番環境のURLを誤って撮影対象に含めていないか（ステージング環境を使うことを推奨）

注意点

  • 認証が必要な画面のスクリーンショットは、テスト用アカウントのCredentialsをGitHub Secretsに保存して利用する。
  • 画像の差分検出にはpixelmatchライブラリが使いやすい。ピクセル差分が小さい場合（フォント変更等）は閾値を設定して誤検出を防ぐ。
  • CI実行時間が長くなる場合は、スクリーンショット更新を週次実行に変更するか、影響を受けるページのみに絞る。
