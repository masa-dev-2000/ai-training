n8nワークフローを自然言語で記述 → JSON自動生成 → 即インポート

★★★ 設定半日〜 ノーコードのさらなる効率化 n8n Claude Code n8n API
← 一覧に戻る

概要

「Gmailで受信したPDFをGoogle Driveに保存してSlackに通知する」のような自然言語でワークフローを記述するだけで、n8nのワークフローJSONをClaude Codeが自動生成し、n8n APIを通じて即インポートできる。

必要な準備

n8n（セルフホスト or n8n Cloud） n8n REST API（有効化済み） Claude Code n8nのノード仕様ドキュメント（参考）

設定手順

  • 1
    n8n APIを有効化する

    n8n設定 → API → API Key を発行。N8N_API_KEY と N8N_BASE_URL を環境変数に設定する。

    N8N_API_KEY=your_api_key N8N_BASE_URL=https://your-n8n.example.com/api/v1
  • 2
    n8nワークフロー生成プロンプトをClaude Codeで設計
    プロンプト例
    以下のワークフローをn8nのJSON形式で生成してください。

ワークフロー名：「請求書PDF処理」

処理内容：
1. Gmailで件名に「請求書」を含むメールを受信したらトリガー
2. 添付PDFファイルをBase64デコード
3. Google Driveの「請求書/YYYY-MM」フォルダに保存
   （フォルダが存在しない場合は作成）
4. Slack #経理チャンネルに「新着請求書：ファイル名」と通知

n8nのノード仕様に準拠したJSON（nodes配列とconnections）を出力してください。
  • 3
    n8n APIでワークフローを自動インポート
    # import_workflow.py（Claude Codeで生成） import requests, json def import_workflow(workflow_json: dict) -> str: response = requests.post( f"{N8N_BASE_URL}/workflows", headers={ "X-N8N-API-KEY": N8N_API_KEY, "Content-Type": "application/json" }, json=workflow_json ) if response.status_code == 200: wf_id = response.json()["id"] return f"ワークフロー作成完了: ID={wf_id}" return f"エラー: {response.text}"
  • 4
    認証情報を手動で設定してアクティブ化

    インポートされたワークフローのGmail/Drive/Slack等の認証情報はn8nのUI上で手動設定が必要。設定後にワークフローをアクティブ化する。

注意点

  • n8nのノード仕様はバージョンによって異なる。Claude Codeに「n8n v1.x系の仕様で生成して」と指定するか、既存ワークフローのJSONを参考として渡すと精度が上がる。
  • 生成されたJSONは必ずn8n UIで動作確認してから本番環境にアクティブ化する。
  • 認証情報（APIキー・OAuth）はn8nのCredentials機能で管理。JSOに直接トークンを含めないこと。
