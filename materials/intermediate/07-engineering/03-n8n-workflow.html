n8nワークフローを自然言語で記述 → JSON自動生成 → 即インポート

★★★ 設定半日〜 ノーコードのさらなる効率化 n8n Claude Code n8n API
← 一覧に戻る

概要

「Gmailで受信したPDFをGoogle Driveに保存してSlackに通知する」のような自然言語でワークフローを記述するだけで、n8nのワークフローJSONをClaude Codeが自動生成し、n8n APIを通じて即インポートできる。

完成イメージ（出力例）

↓「GmailでPDFを受信したらSlackに通知する」という自然言語指示から生成されたn8n JSONの例（抜粋）

設定例（ケース別）

Slack通知ワークフロー — プロンプト例
以下のワークフローをn8nのJSON形式で生成してください。

ワークフロー名：「フォーム送信 → Slack通知」

処理内容：
1. Webhookノード（POST /webhook/contact）でフォーム送信を受け取る
2. 受信データのバリデーション（name・email・message が空でないか確認）
3. Slack #問い合わせチャンネルに以下の形式で通知：
   「新着問い合わせ
   名前：{{name}}
   メール：{{email}}
   内容：{{message}}」
4. Webhookに 200 OK レスポンスを返す

n8n v1.x 系の仕様に準拠したJSON（nodes配列・connections・settings）を出力してください。
既存ワークフローがある場合は添付するのでそれに準拠させてください。
データ同期ワークフロー — プロンプト例
以下のワークフローをn8nのJSON形式で生成してください。

ワークフロー名：「顧客データ Salesforce → Notion 同期」

処理内容：
1. Salesforce のトリガー（新規リードが登録されたとき）
2. Salesforce MCPで新規リードの詳細情報を取得
3. Notion データベース「顧客リスト」に以下のプロパティでページを作成：
   - 名前（タイトル）
   - 会社名（テキスト）
   - メール（メール）
   - 登録日（日付）
   - ステータス（セレクト）：「未対応」で固定
4. Slack #営業チャンネルに「新規リード追加: {{リード名}}」と通知

重複チェック：メールアドレスで既存ページを検索し、存在する場合はスキップする。
定期実行ワークフロー — プロンプト例
以下のワークフローをn8nのJSON形式で生成してください。

ワークフロー名：「週次レポート自動送信」

処理内容：
1. スケジュールトリガー（毎週月曜 9:00）
2. Google Sheets「売上データ」シートから過去7日分のデータを取得
3. データを集計（合計・平均・前週比）
4. Claudeノード（または HTTPリクエスト）を使ってサマリー文章を生成
5. 指定のメールアドレス宛にHTMLメールで送信（件名：「週次売上レポート YYYY/MM/DD」）
6. 送信ログを Google Sheets「送信履歴」シートに追記

エラーが発生した場合は Slack #エラー通知チャンネルにアラートを送る。

必要な準備

n8n（セルフホスト or n8n Cloud） n8n REST API（有効化済み） Claude Code n8nのノード仕様ドキュメント（参考）

設定手順

  • 1
    n8n APIを有効化する

    n8n設定 → API → API Key を発行。N8N_API_KEY と N8N_BASE_URL を環境変数に設定する。

    N8N_API_KEY=your_api_key N8N_BASE_URL=https://your-n8n.example.com/api/v1
  • 2
    n8nワークフロー生成プロンプトをClaude Codeで設計
    プロンプト例
    以下のワークフローをn8nのJSON形式で生成してください。

ワークフロー名：「請求書PDF処理」

処理内容：
1. Gmailで件名に「請求書」を含むメールを受信したらトリガー
2. 添付PDFファイルをBase64デコード
3. Google Driveの「請求書/YYYY-MM」フォルダに保存
   （フォルダが存在しない場合は作成）
4. Slack #経理チャンネルに「新着請求書：ファイル名」と通知

n8nのノード仕様に準拠したJSON（nodes配列とconnections）を出力してください。
  • 3
    n8n APIでワークフローを自動インポート
    # import_workflow.py（Claude Codeで生成） import requests, json def import_workflow(workflow_json: dict) -> str: response = requests.post( f"{N8N_BASE_URL}/workflows", headers={ "X-N8N-API-KEY": N8N_API_KEY, "Content-Type": "application/json" }, json=workflow_json ) if response.status_code == 200: wf_id = response.json()["id"] return f"ワークフロー作成完了: ID={wf_id}" return f"エラー: {response.text}"
  • 4
    認証情報を手動で設定してアクティブ化

    インポートされたワークフローのGmail/Drive/Slack等の認証情報はn8nのUI上で手動設定が必要。設定後にワークフローをアクティブ化する。

確認チェックリスト

  • 生成されたJSONをn8n UIにインポートし、ノードのエラー（赤いマーク）がないか確認したか
  • Gmail・Slack・Notion等の認証情報（Credentials）をn8n UI上で設定したか
  • ワークフローをテスト実行し、意図した動作になることを確認したか
  • 本番データ・本番サービスに影響するワークフローはテスト環境で動作確認してからアクティブ化したか
  • APIキー・OAuthトークン等の機密情報がJSONに直接含まれていないか
  • エラーハンドリングノード（エラー時のSlack通知等）を設定したか
  • n8nのバージョンを指定してJSONを生成したか（バージョン不一致は動作不良の原因になる）

注意点

  • n8nのノード仕様はバージョンによって異なる。Claude Codeに「n8n v1.x系の仕様で生成して」と指定するか、既存ワークフローのJSONを参考として渡すと精度が上がる。
  • 生成されたJSONは必ずn8n UIで動作確認してから本番環境にアクティブ化する。
  • 認証情報（APIキー・OAuth）はn8nのCredentials機能で管理。JSOに直接トークンを含めないこと。
