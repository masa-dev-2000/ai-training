SNSブランド評判監視 → ネガティブ急増でSlack緊急通知

★★★ 設定半日〜 24時間監視の自動化 X API v2 Claude Code Slack MCP
← 一覧に戻る

概要

X（旧Twitter）等のSNSでブランド名・商品名への言及を定期監視し、Claudeが感情分析（ポジティブ・ネガティブ・中立）を実施。ネガティブ投稿が急増した際にSlackの緊急チャンネルへ即座に通知する炎上検知システム。

必要な準備

X Developer Portal（Basic以上のプラン） X API v2 Bearerトークン Slack Incoming Webhook URL Python（tweepy, anthropic）

設定手順

  • 1
    X APIの認証情報を設定
    X_BEARER_TOKEN=your_bearer_token SLACK_WEBHOOK_URL=https://hooks.slack.com/services/xxx/yyy/zzz ANTHROPIC_API_KEY=your_api_key
  • 2
    監視キーワードを定義する
    # keywords.json { "brand_keywords": ["会社名", "商品名A", "商品名B", "#ブランドハッシュタグ"], "alert_threshold": 5, "check_interval_minutes": 30, "alert_channel": "#緊急-炎上対応" }
  • 3
    感情分析スクリプトを作成（Claude Codeで生成）
    # brand_monitor.py（抜粋） import tweepy, anthropic, json, requests def analyze_sentiment(tweets: list[str]) -> dict: client = anthropic.Anthropic() text = "\n".join([f"- {t}" for t in tweets]) response = client.messages.create( model="claude-haiku-4-5-20251001", max_tokens=300, messages=[{"role": "user", "content": f"以下のツイートを感情分析してください。ポジティブ/ネガティブ/中立の件数と、\n" f"ネガティブが多い場合は主な批判内容をまとめてください。\n\n{text}"}] ) return response.content[0].text def alert_slack(message: str, channel: str): requests.post(SLACK_WEBHOOK_URL, json={"text": message, "channel": channel})
  • 4
    閾値と通知条件を設定する

    30分以内にネガティブ投稿が5件を超えたら緊急通知、1時間で10件超えたらエスカレーション（電話・SMS）を発動するロジックを設定。

  • 5
    cronまたはCloud Schedulerで定期実行
    # 30分ごとに実行 */30 * * * * python brand_monitor.py

注意点

  • X API v2は無料プランでは月15万ツイートの取得制限がある。有料プランへの移行が必要な場合がある。
  • 感情分析の精度は100%ではない。皮肉・冗談・引用ツイートは誤分類されやすい。誤検知を減らすため閾値は実際の投稿量を見て調整する。
  • 炎上と判断した場合の対応フロー（担当者・返答方針・承認プロセス）を事前に決めておく。システムはあくまで検知まで。
