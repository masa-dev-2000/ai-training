SNSブランド評判監視 → ネガティブ急増でSlack緊急通知

★★★ 設定半日〜 24時間監視の自動化 X API v2 Claude Code Slack MCP
← 一覧に戻る

概要

X（旧Twitter）等のSNSでブランド名・商品名への言及を定期監視し、Claudeが感情分析（ポジティブ・ネガティブ・中立）を実施。ネガティブ投稿が急増した際にSlackの緊急チャンネルへ即座に通知する炎上検知システム。

完成イメージ（出力例）

↓ ネガティブ投稿が閾値を超えた際のSlack通知メッセージ例

Slack #緊急-炎上対応 チャンネルへの通知メッセージ
🚨 【ブランド監視アラート】ネガティブ急増を検知

監視キーワード：「〇〇サービス」「#〇〇不具合」
検知時刻：2026-05-15 14:32

■ 直近30分の集計
・取得ツイート数：47件
・ポジティブ：8件（17%）
・ネガティブ：31件（66%）← 閾値5件を超過（31件）
・中立：8件（17%）

■ 主な批判内容（Claudeによる要約）
過去30分に急増したネガティブ投稿の主な内容：
1. ログインできないという報告が多数（19件）
2. アプリが落ちるという声（8件）
3. 対応が遅いとの不満（4件）

■ 代表的な投稿（上位3件）
・「〇〇サービスまた落ちてる…仕事にならない」（RT: 42）
・「ログイン画面から先に進めない。サポートに連絡したが繋がらない」（RT: 28）
・「〇〇の障害、もう3回目。さすがに解約考える」（RT: 15）

対応担当：@marketing-team @support-team
エスカレーション：10件超過で電話連絡を発動予定

必要な準備

X Developer Portal（Basic以上のプラン） X API v2 Bearerトークン Slack Incoming Webhook URL Python（tweepy, anthropic）

設定手順

  • 1
    X APIの認証情報を設定
    X_BEARER_TOKEN=your_bearer_token SLACK_WEBHOOK_URL=https://hooks.slack.com/services/xxx/yyy/zzz ANTHROPIC_API_KEY=your_api_key
  • 2
    監視キーワードを定義する
    # keywords.json { "brand_keywords": ["会社名", "商品名A", "商品名B", "#ブランドハッシュタグ"], "alert_threshold": 5, "check_interval_minutes": 30, "alert_channel": "#緊急-炎上対応" }
  • 3
    感情分析スクリプトを作成（Claude Codeで生成）
    # brand_monitor.py（抜粋） import tweepy, anthropic, json, requests def analyze_sentiment(tweets: list[str]) -> dict: client = anthropic.Anthropic() text = "\n".join([f"- {t}" for t in tweets]) response = client.messages.create( model="claude-haiku-4-5-20251001", max_tokens=300, messages=[{"role": "user", "content": f"以下のツイートを感情分析してください。ポジティブ/ネガティブ/中立の件数と、\n" f"ネガティブが多い場合は主な批判内容をまとめてください。\n\n{text}"}] ) return response.content[0].text def alert_slack(message: str, channel: str): requests.post(SLACK_WEBHOOK_URL, json={"text": message, "channel": channel})
  • 4
    閾値と通知条件を設定する

    30分以内にネガティブ投稿が5件を超えたら緊急通知、1時間で10件超えたらエスカレーション（電話・SMS）を発動するロジックを設定。

  • 5
    cronまたはCloud Schedulerで定期実行
    # 30分ごとに実行 */30 * * * * python brand_monitor.py

プロンプト / 設定例（ケース別）

ブランド名監視 — 感情分析プロンプト
以下のX投稿リストを感情分析してください。

【投稿リスト】
（tweepyで取得したツイートテキストを貼り付け）

分析してほしいこと：
1. ポジティブ・ネガティブ・中立の件数と割合
2. ネガティブ投稿の主な批判内容（上位3つのテーマにまとめる）
3. 緊急度の判定（低/中/高/緊急）
   ・緊急：製品不具合・サービス障害・法的リスク
   ・高：大量クレーム・インフルエンサーによる批判
   ・中：一般的な不満
   ・低：軽微なネガティブ
4. 推奨する初動対応（1〜2文で）

JSON形式で出力してください。
競合監視 — 競合への評価収集プロンプト
以下の競合ブランドに関するX投稿を分析してください。

監視対象：「競合社名A」「競合商品B」

分析内容：
1. 競合への感情傾向（ポジティブ/ネガティブ/中立の割合）
2. よく挙げられている競合の強み（ポジティブ投稿から）
3. よく挙げられている競合の弱点・不満（ネガティブ投稿から）
4. 自社が差別化できるチャンスポイント
5. 注目すべき投稿（RT数の多いもの上位3件）

競合の弱点は自社の強みとして訴求できる可能性があるため、
具体的な改善提案も含めてください。
キーワード監視 — 業界トレンド把握プロンプト
以下のキーワードに関するX投稿を分析して業界トレンドをまとめてください。

監視キーワード：「〇〇業界」「〇〇の課題」「〇〇 困った」

分析内容：
1. 最も多く言及されているトピックのランキング（上位10個）
2. ユーザーが抱えている悩み・課題（自社製品のヒントになるもの）
3. 急上昇中のトピック（先週比で急増しているワード）
4. インフルエンサー（フォロワー5000以上）の主な発言傾向
5. 自社コンテンツ・製品開発へのヒント（3点）

週次でこの分析を実施し、Google Sheetsに蓄積してトレンド変化を追跡します。

確認チェックリスト

  • X API Bearerトークン・Slack Webhook URLなどの認証情報を .env ファイルに保存し、Gitに含まれないよう設定したか
  • 監視キーワードに自社ブランドの表記ゆれ（略称・誤字・カタカナ・ひらがな）を含めたか
  • ネガティブ検知時の対応フロー（担当者・返答方針・承認プロセス）を事前にチームで決めたか
  • 感情分析の精度をテストデータで確認し、誤検知の傾向を把握したか（皮肉・冗談の誤分類など）
  • Slackのアラートが正しいチャンネルに届くかテスト送信で確認したか
  • X APIの月次ツイート取得上限（無料プランで15万件）を把握し、過剰リクエストにならないよう設定したか
  • 閾値（何件でアラートを発動するか）を実際の投稿量に合わせて調整したか

注意点

  • X API v2は無料プランでは月15万ツイートの取得制限がある。有料プランへの移行が必要な場合がある。
  • 感情分析の精度は100%ではない。皮肉・冗談・引用ツイートは誤分類されやすい。誤検知を減らすため閾値は実際の投稿量を見て調整する。
  • 炎上と判断した場合の対応フロー（担当者・返答方針・承認プロセス）を事前に決めておく。システムはあくまで検知まで。
