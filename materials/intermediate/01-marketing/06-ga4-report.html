GA4×BigQuery 週次・月次広告レポート定期自動生成

★★★ 設定半日〜 3〜5時間 → 7分 GA4 BigQuery Claude Code
← 一覧に戻る

概要

GA4とBigQueryを連携し、Claude CodeがSQLクエリを自動生成・実行してデータを取得。AIが分析コメントを付与したレポートをGoogle Sheets / PDFで定期出力する。手動で3〜5時間かかっていたレポート作業が7分で完了する。

必要な準備

GA4プロパティ（BigQueryエクスポート有効化） Google Cloud Platform アカウント BigQuery API有効化 サービスアカウント認証情報（JSON） Python（google-cloud-bigquery, gspread）

設定手順

  • 1
    GA4のBigQueryエクスポートを有効化

    GA4管理画面 → 管理 → BigQueryのリンク → BigQueryプロジェクトを指定してエクスポートを有効化。毎日イベントデータが自動でBigQueryに蓄積される。

  • 2
    サービスアカウントを作成して認証設定
    export GOOGLE_APPLICATION_CREDENTIALS="path/to/service-account.json"
  • 3
    レポート用SQLクエリをClaude Codeで生成
    プロンプト例
    GA4のBigQueryテーブル（`project.analytics_XXXXXX.events_*`）から以下の週次レポートを生成するSQLクエリとPythonスクリプトを作成してください。

取得指標：
- 週次セッション数・ユーザー数・直帰率
- チャネル別コンバージョン率（CVR）
- ページ別滞在時間上位10
- 前週比の増減率

出力：Google Sheetsのシートに以下の形式で書き込む
・サマリーシート（数値と前週比）
・チャネル別シート
・コメントシート（AIによる改善提案）
  • 4
    AIコメント生成を追加

    数値データを取得した後、Claude CodeのAPIを使って改善提案コメントを自動生成する部分をスクリプトに追加する。

    # report_generator.py（抜粋） import anthropic def generate_ai_comment(metrics_dict): client = anthropic.Anthropic() message = client.messages.create( model="claude-opus-4-7", max_tokens=500, messages=[{ "role": "user", "content": f"以下の広告指標を分析して改善提案を3点述べてください：\n{metrics_dict}" }] ) return message.content[0].text
  • 5
    定期実行を設定

    Cloud Schedulerまたはcronで毎週月曜AM8時・毎月1日に自動実行するよう設定。Slackにレポートリンクを通知するとさらに便利。

    # crontab 0 8 * * 1 python report_generator.py --mode=weekly 0 8 1 * * python report_generator.py --mode=monthly

注意点

  • BigQueryは従量課金制。クエリのデータ処理量に応じて費用が発生する。無料枠（月10GB）を超えないようWHERE句で日付絞り込みを必ず行う。
  • GA4のBigQueryエクスポートは有効化から最初のデータが届くまで24〜48時間かかる。
  • レポートに含まれる指標の定義をチームで統一しておく。GA4とBigQueryで数値が一致しない場合がある（サンプリング等の影響）。
