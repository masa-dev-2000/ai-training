GA4×BigQuery 週次・月次広告レポート定期自動生成

★★★ 設定半日〜 3〜5時間 → 7分 GA4 BigQuery Claude Code
← 一覧に戻る

概要

GA4とBigQueryを連携し、Claude CodeがSQLクエリを自動生成・実行してデータを取得。AIが分析コメントを付与したレポートをGoogle Sheets / PDFで定期出力する。手動で3〜5時間かかっていたレポート作業が7分で完了する。

必要な準備

GA4プロパティ（BigQueryエクスポート有効化） Google Cloud Platform アカウント BigQuery API有効化 サービスアカウント認証情報（JSON） Python（google-cloud-bigquery, gspread）

設定手順

  • 1
    GA4のBigQueryエクスポートを有効化

    GA4管理画面 → 管理 → BigQueryのリンク → BigQueryプロジェクトを指定してエクスポートを有効化。毎日イベントデータが自動でBigQueryに蓄積される。

  • 2
    サービスアカウントを作成して認証設定
    export GOOGLE_APPLICATION_CREDENTIALS="path/to/service-account.json"
  • 3
    レポート用SQLクエリをClaude Codeで生成
    プロンプト例
    GA4のBigQueryテーブル（`project.analytics_XXXXXX.events_*`）から以下の週次レポートを生成するSQLクエリとPythonスクリプトを作成してください。

取得指標：
- 週次セッション数・ユーザー数・直帰率
- チャネル別コンバージョン率（CVR）
- ページ別滞在時間上位10
- 前週比の増減率

出力：Google Sheetsのシートに以下の形式で書き込む
・サマリーシート（数値と前週比）
・チャネル別シート
・コメントシート（AIによる改善提案）
  • 4
    AIコメント生成を追加

    数値データを取得した後、Claude CodeのAPIを使って改善提案コメントを自動生成する部分をスクリプトに追加する。

    # report_generator.py（抜粋） import anthropic def generate_ai_comment(metrics_dict): client = anthropic.Anthropic() message = client.messages.create( model="claude-opus-4-7", max_tokens=500, messages=[{ "role": "user", "content": f"以下の広告指標を分析して改善提案を3点述べてください：\n{metrics_dict}" }] ) return message.content[0].text
  • 5
    定期実行を設定

    Cloud Schedulerまたはcronで毎週月曜AM8時・毎月1日に自動実行するよう設定。Slackにレポートリンクを通知するとさらに便利。

    # crontab 0 8 * * 1 python report_generator.py --mode=weekly 0 8 1 * * python report_generator.py --mode=monthly

プロンプト / 設定例（ケース別）

週次レポート — SQLクエリ生成プロンプト
GA4のBigQueryテーブルから先週1週間分のデータを集計する週次レポート用SQLを生成してください。

テーブル名：`my-project.analytics_123456789.events_*`
対象期間：自動で先週月曜〜日曜を計算すること

取得する指標：
・セッション数、ユーザー数、新規ユーザー数
・チャネル別セッション数とCVR
・コンバージョンイベント（purchase / sign_up）の件数
・前週比（前の週と比較する副問い合わせも含める）
・直帰率（engaged_session_rateの逆数）

出力形式：Pythonスクリプトとして完成させる。
結果はGoogle Sheetsの指定シートに書き込むこと。
月次レポート — 経営層向けサマリープロンプト
GA4のBigQueryから先月1ヶ月分のデータを集計する月次レポートを生成してください。

対象期間：自動で先月1日〜末日を計算すること

取得指標（週次より詳細）：
・月次セッション・ユーザー・CV・売上（ecommerce）
・チャネル別の費用対効果（広告費用データはCSVで別途入力）
・上位10ページのパフォーマンス
・デバイス別（PC/スマホ/タブレット）の比較
・前月比・前年同月比

出力形式：
・Google Sheetsにグラフ用データを整形して書き込む
・経営層向けサマリー（250字以内の日本語テキスト）をAIに生成させる
・PDFエクスポート用のHTMLレポートも生成する
特定指標の深掘り — CVR低下調査プロンプト
GA4のBigQueryデータを使って、今週CVRが低下した原因を深掘り調査してください。

調査対象期間：今週（vs 先週）
問題：全体CVRが先週比-0.5ptと低下

調査観点：
1. チャネル別のCVR変化（どのチャネルで下がっているか）
2. デバイス別のCVR変化（スマホで下がっているか等）
3. 流入ページ別のCVR変化（特定ページで離脱が増えていないか）
4. 時間帯別のパターン（特定の時間帯に集中していないか）
5. 新規 vs リピーターのCVR比較

各観点のSQLを生成し、分析結果と仮説をコメントとして出力してください。

確認チェックリスト

  • GA4のBigQueryエクスポートが有効化されており、データが正しく蓄積されているか確認したか（有効化後24〜48時間で最初のデータが届く）
  • SQLクエリにWHERE句での日付絞り込みを追加し、BigQueryの処理量（コスト）を抑えているか
  • サービスアカウントのJSONキーが .gitignore に含まれており、リポジトリに含まれないよう設定したか
  • 生成されたレポートの数値をGA4管理画面の数値と突き合わせて一致しているか確認したか
  • Slackへのレポートリンク通知を設定した場合、テスト通知が正しく届くか確認したか
  • cronまたはCloud Schedulerで定期実行が設定済みで、初回テスト実行が正常に完了したか

注意点

  • BigQueryは従量課金制。クエリのデータ処理量に応じて費用が発生する。無料枠（月10GB）を超えないようWHERE句で日付絞り込みを必ず行う。
  • GA4のBigQueryエクスポートは有効化から最初のデータが届くまで24〜48時間かかる。
  • レポートに含まれる指標の定義をチームで統一しておく。GA4とBigQueryで数値が一致しない場合がある（サンプリング等の影響）。
