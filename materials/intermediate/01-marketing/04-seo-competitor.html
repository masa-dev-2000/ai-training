SEO競合調査 Hタグ抽出→記事構成案自動生成

★★★ 設定半日〜 競合10サイト→構成案まで自動 Claude Code Python Google Sheets API
← 一覧に戻る

概要

ターゲットキーワードで検索上位10サイトのH1〜H3タグを自動抽出し、Claude CodeでAI分析。共通トピック・競合が網羅していない空白領域を特定して記事構成案をスプレッドシートに保存する。

完成イメージ（出力例）

↓ キーワード「副業 確定申告」で競合5社を調査した場合の構成案出力例

Claude Code が生成した記事構成案
■ 競合カバレッジ分析（上位5サイト横断）

【全サイト共通トピック（空白なし）】
・確定申告が必要な所得金額の基準（5/5サイト）
・青色申告 vs 白色申告の違い（5/5サイト）
・必要書類の一覧（4/5サイト）

【差別化チャンス（2サイト以下しか触れていない空白領域）】
・副業ごとの経費の扱い方（ライター・ハンドメイド・プログラミング別）
・ふるさと納税との組み合わせ節税術
・会社員がバレずに副業収入を申告するポイント

■ 提案記事構成案
H1: 副業の確定申告を完全解説｜いくらから？必要書類・書き方まで

  H3: 給与所得以外の所得が20万円を超えたら申告義務
  H3: 住民税は金額に関わらず申告が必要な場合も

H2: 副業の種類別・経費の計上ルール（差別化ポイント）
  H3: ライター・ブロガーの場合
  H3: ハンドメイド販売・せどりの場合
  H3: プログラミング・エンジニア副業の場合

  H3: 青色申告のメリット（65万円控除）

必要な準備

Python（requests, BeautifulSoup, gspread） Google Sheets API 認証情報（サービスアカウント） Claude Code

設定手順

  • 1
    競合URLリストを準備する

    Google検索で対象キーワードを検索し、上位10サイトのURLを urls.txt に1行1URLで保存する（広告・NAP除く）。

  • 2
    HタグスクレイパーをClaude Codeで生成
    プロンプト例
    urls.txtに記載されたURLリストから各ページのH1〜H3タグをスクレイピングするPythonスクリプトを作成してください。

出力形式：
- サイトごとにH1/H2/H3を階層表示
- 全サイト横断で登場頻度の高い見出しワードをランキング
- 結果をcompetitor_headings.csvに保存

ライブラリ：requests, BeautifulSoup4
エラー処理：アクセス失敗したURLはスキップしてログに記録
  • 3
    スクリプトを実行してCSVを取得
    pip install requests beautifulsoup4 gspread python scrape_headings.py
  • 4
    Claude Codeで競合分析と構成案を生成
    分析プロンプト例
    competitor_headings.csvを分析して以下を教えてください：

1. 競合10サイトが共通して網羅しているトピック（上位20個）
2. 半数以下しか触れていない「空白トピック」（差別化チャンス）
3. 検索意図に基づく記事構成案（H1・H2・H3の階層で提案）

対象キーワード：「〇〇」
想定読者：〇〇（職種・レベル）
記事の目標：〇〇（CV・情報提供・比較等）
  • 5
    構成案をGoogle Sheetsに保存

    Claude Codeに「Google Sheetsに書き込むコードも追加して」と指示し、分析結果を自動でスプレッドシートに格納する。

プロンプト / 設定例（ケース別）

以下のURLのページを詳しく分析してください。

分析項目：
1. H1〜H3の見出し構成（階層ごとに表示）
2. 各セクションのおおよその文字数
3. 内部リンク・外部リンクの傾向
4. 想定される検索意図と記事の狙い
5. 私たちが参考にすべき点・差別化できる点

対象キーワード：「〇〇」
自社の強み・差別化したい点：〇〇
上位5社一括比較 — 横断分析プロンプト
competitor_headings.csvに記録された上位5サイトのHタグを横断分析してください。

分析内容：
1. カバレッジ表（H2レベルのトピック × 各サイト）
   ・○：詳しく扱っている
   ・△：軽く触れている
   ・×：扱っていない
2. 全サイト共通の必須トピック（読者が期待する内容）
3. 差別化余地のある空白トピック（上位3つ）
4. 見出しのパターン・傾向（数字入り・問いかけ型・ハウツー型など）
5. 推奨記事構成案（H1〜H3まで完全版）

対象キーワード：「〇〇」
目標文字数：〇〇字
記事構成案のみ — 直接生成プロンプト
以下の条件で記事の構成案をH1〜H3まで提案してください。

対象キーワード：「〇〇」
検索意図：〇〇（情報収集 / 比較検討 / 購入 / 操作方法など）
想定読者：〇〇（初心者 / 中級者 / 専門家など）
目標文字数：〇〇字
記事の目的：〇〇（CVR向上 / 検索流入 / ブランディング）

参考：以下の競合サイトが共通して扱っているトピック
（competitor_headings.csvの分析結果を貼り付け）

競合が触れていない切り口を盛り込み、独自性を出してください。

確認チェックリスト

  • 対象サイトのrobots.txtを確認し、スクレイピングが禁止されていないか確認したか
  • アクセス間隔を1〜2秒空ける設定をスクリプトに入れたか
  • JavaScriptで動的に描画されるサイトが含まれていないか確認したか（含む場合はPlaywright使用を検討）
  • 取得したCSVのデータを目視確認し、Hタグが正しく取得されているか確認したか
  • 構成案の「空白トピック」が本当に読者のニーズに合致するか検証したか
  • Google Sheets APIのサービスアカウントにスプレッドシートの編集権限を付与したか

注意点

  • スクレイピングは対象サイトの利用規約を確認すること。robots.txtを遵守する。
  • JavaScriptで動的に描画されるサイトはBeautifulSoupでは取得できない。Playwrightを使う必要がある。
  • User-Agentを設定しないとブロックされるサイトがある。適切なUser-Agentを設定し、アクセス間隔も1〜2秒あける。
  • 競合分析は月1回程度の実施で十分。毎日実行するとIPブロックのリスクがある。
