商談メモ → CRM自動入力 ＋ 次アクション提案

★★★ 設定半日〜 商談後の入力作業を自動化 HubSpot API Claude Code
← 一覧に戻る

概要

商談後に書いたメモ（箇条書きでもOK）をClaude Codeに渡すと、構造化してHubSpot / SalesforceのCRMに自動入力し、商談フェーズと会話内容に基づいて次のアクションを提案する。商談後の入力作業（平均20〜30分）を3分に短縮する。

必要な準備

HubSpot アカウント（無料プランでも可） HubSpot Private App APIキー Claude Code Python（hubspot-api-client）

設定手順

  • 1
    HubSpot Private App を作成してAPIキーを取得

    HubSpot → 設定 → インテグレーション → プライベートアプリ → アプリ作成。スコープ：crm.objects.contacts.write、crm.objects.deals.write、crm.objects.notes.write

  • 2
    MCPサーバーまたはスクリプトを作成（Claude Codeで生成）
    プロンプト例
    HubSpot APIを使って商談メモから以下を自動処理するPythonスクリプトを作成してください。

処理内容：
1. 商談メモのテキストから構造化データを抽出
   （会社名・担当者・課題・予算・決裁者・競合・次回アクション・確度）
2. HubSpotの対応するコンタクト・案件を検索or作成
3. アクティビティノートとして記録
4. 案件フェーズを更新
5. 次のアクション候補3つを提案（内容・期限・優先度付き）

HubSpot APIキー：環境変数から取得
  • 3
    スキルコマンド化する
    # .claude/skills/crm-update.md --- name: crm-update description: 商談メモをCRMに自動入力して次アクションを提案する --- ユーザーの商談メモからcrm_updater.pyを実行し、 HubSpotへの自動入力と次アクション提案を行ってください。

    実行例

プロンプト / 設定例（ケース別）

初回商談メモ — 新規コンタクトとして登録
/crm-update

【商談種別】初回訪問
【日時】2026年5月15日 14:00〜15:30
【先方】株式会社〇〇 / 営業部 田中部長（tanaka@example.co.jp）/ 製造業・従業員500名
【自社参加者】山田（営業）

【商談内容メモ】
- 展示会で名刺交換、今回が初訪問
- 現状の課題：受注管理がExcelで属人化、担当者不在時に業務が止まる
- 現在のシステム：〇〇ERPを使用（2017年導入、刷新を検討中）
- 予算感：まだ決まっていない。まずは比較検討フェーズ
- 決裁プロセス：田中部長が稟議を上げ、取締役会で承認
- 次回：資料送付後、2週間後に再訪問を提案。先方も同意

→ HubSpotに新規コンタクト・新規案件として登録し、
  フェーズを「初回接触」に設定してください。

【商談種別】フォローアップ（電話）
【日時】2026年5月15日 11:00〜11:20
【先方】株式会社〇〇 / 鈴木課長（既存コンタクト）
【自社参加者】山田

【通話内容メモ】
- 先週送付した提案書について感触を確認
- 全体的には好評。特に自動化機能に興味あり
- 懸念点：既存ERPとの連携が可能かどうか確認したい

クロージング前チェック — 受注直前の最終確認
/crm-update

【商談種別】クロージング商談
【日時】2026年5月15日 16:00〜17:00
【先方】株式会社〇〇 / 田中社長・鈴木部長（2名）
【自社参加者】山田（営業）・佐藤（マネージャー）

【商談内容メモ】
- 社内稟議が通過した。来月から契約したい
- 契約金額：年額180万円（12ヶ月一括払い）
- 開始時期：6月1日を希望
- 契約書のドラフトを来週中に送付してほしい
- 懸案だったERP連携は技術チームで対応可能と確認済み

→ 案件フェーズを「クロージング」に変更し、
  契約金額・開始予定日を案件情報に更新してください。
  また、契約書送付・導入準備の次アクションを作成してください。

入力前チェックリスト

  • 既存コンタクトの重複がないか（同じ会社・担当者が二重登録になっていないか）
  • 予算・金額の数値がメモの内容と一致しているか（AIが概算した数値になっていないか）
  • 確度（A/B/C）の判定がメモの内容に基づいているか
  • 次のアクションの期日が現実的かつ抜け漏れがないか
  • 先方が話した内容と、担当者の推測が混在していないか
  • 顧客の個人情報（メールアドレス・電話番号）が正確に入力されているか
  • CRMへの反映後、HubSpot画面で実際に確認が取れているか

注意点

  • 顧客の個人情報・商談金額はClaude Codeのプロジェクト設定でデータ保持をオフにしてから使用する。
  • HubSpotの既存データと重複しないよう、コンタクト検索ロジック（メールアドレス・会社名）を丁寧に実装する。
  • Salesforceを使う場合はsimple-salesforceライブラリを使用。APIスコープはHubSpotと異なるため調整が必要。
