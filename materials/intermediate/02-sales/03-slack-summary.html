Slack MCP経由で先週のsalesチャンネル議論を自動要約

★★ 設定1〜2時間 週次営業サマリーを自動化 Slack MCP Claude Code
← 一覧に戻る

概要

Slack MCPを通じてsalesチャンネルの過去1週間のメッセージを取得し、Claudeが自動要約。商談進捗・課題・決定事項・次週のアクションをまとめた週次営業レポートをSlackに自動投稿する。

必要な準備

Slack ワークスペースの管理権限 Slack APIトークン（channels:history スコープ） Claude Code

設定手順

  • 1
    Slack APIトークンを取得する

    api.slack.com → Your Apps → アプリ作成 → OAuth & Permissions で以下のスコープを追加：channels:history、chat:write、users:read

  • 2
    Slack MCPをClaude Codeに追加
    # .claude/settings.json { "mcpServers": { "slack": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-slack"], "env": { "SLACK_BOT_TOKEN": "xoxb-your-token", "SLACK_TEAM_ID": "T0000000" } } } }
  • 3
    週次サマリースキルを作成
    # .claude/skills/weekly-sales-summary.md --- name: weekly-sales-summary description: salesチャンネルの先週の議論を要約してレポートを投稿する --- Slack MCPを使って#salesチャンネルの過去7日間のメッセージを取得し、 以下の形式で要約して#sales-reportチャンネルに投稿してください。 ## 今週の営業サマリー（〇月〇日〜〇日） ### 商談進捗 - 新規商談：〇件 - 受注：〇件（〇〇万円） - 失注：〇件 ### 主要な議論・決定事項 （箇条書き3〜5点） ### 来週のアクション （担当者名付きで箇条書き）
  • 4
    毎週月曜朝に自動実行する設定
    # crontab（毎週月曜AM9時に実行） 0 9 * * 1 claude --skill weekly-sales-summary

注意点

  • プライベートチャンネルのメッセージを取得するには、ボットをチャンネルに招待する必要がある。
  • Slackメッセージには顧客名・金額・個人情報が含まれることがある。要約結果の共有範囲を限定する。
  • スレッド内のメッセージは別途取得が必要。重要な議論がスレッドに流れやすいため、スレッド取得も追加するとよい。
