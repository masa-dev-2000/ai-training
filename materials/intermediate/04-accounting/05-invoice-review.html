概要

月間100〜200枚の請求書確認作業を自動化するパイプラインを構築する。受信したPDFから必要情報を抽出・検証し、問題のないものは自動承認キューへ、疑問点があるものだけを担当者に振り分けることで87%の時間削減を実現する。

自動化の対象業務

# 従来の手作業ステップ（1枚あたり約2〜3分） 1. メール受信 → 添付PDFを開く 2. 発行者名・金額・支払期日を確認 3. 過去の取引データと照合 4. 消費税の計算が正しいか確認 5. インボイス登録番号の確認 6. 支払条件（〇日払い）の確認 7. 会計ソフトへの入力 8. 承認者へ回覧 # 自動化後のステップ（担当者は3.と7.のみ） 1〜6: AI自動チェック 7: 問題なし → 自動承認キュー / 問題あり → 担当者に通知 8: 担当者確認後、会計ソフトに一括インポート

設定手順

  • 1
    請求書受信フォルダを設定

    専用メールアドレス（例：invoice@company.com）を作成し、受信した添付PDFを自動で処理フォルダに保存するフィルタを設定する（Gmailフィルタ → Google Drive保存）。

  • 2
    バッチ処理スクリプトを作成（Claude Codeで生成）
    # invoice_processor.py（抜粋） import pdfplumber, anthropic from pathlib import Path def process_invoices(folder: str): results = [] for pdf_file in Path(folder).glob("*.pdf"): text = extract_text(pdf_file) data = structure_invoice(text) # Claudeで構造化 checks = run_checks(data) # 7項目チェック results.append({ "file": pdf_file.name, "data": data, "status": "OK" if all(checks.values()) else "要確認", "issues": [k for k, v in checks.items() if not v] }) return results
  • 3
    承認キューUIを用意（任意）

    問題なしの請求書一覧を表示するシンプルなHTMLページを作成し、担当者が一括確認・承認できるUIを提供する。

  • 4
    会計ソフトへの一括インポートを設定

    承認済みデータをCSVに変換し、会計ソフトのインポート機能で一括登録する。freeeはCSVインポート対応。

設定例（ケース別）

Gmail自動フィルタ設定 — 受信した請求書PDFを自動振り分け
Gmailで受信した請求書メールを自動でフィルタリングし、
添付PDFをGoogle Driveの処理フォルダに保存するGASを作成してください。

// Google Apps Script（GAS）の設定例
function processInvoiceEmails() {
  const threads = GmailApp.search(
    'subject:(請求書 OR invoice) has:attachment filename:pdf'
  );
  threads.forEach(thread => {
    thread.getMessages().forEach(msg => {
      msg.getAttachments().forEach(att => {
        if (att.getContentType() === 'application/pdf') {
          DriveApp.getFolderById(FOLDER_ID).createFile(att);
          msg.markRead();
          msg.moveToTrash(); // 処理済みをアーカイブ
        }
      });
    });
  });
}

- AIチェック完了 → ステータスに応じてフォルダ移動
- 承認操作後 → 04_承認済みに移動 + 移動日をファイル名に追記
承認フロー設定 — 金額別に承認ルートを自動分岐
請求書の金額によって承認ルートを変えるフロー設定を実装してください。

# 承認ルール（config.yaml）
approval_rules:
  - max_amount: 100000      # 10万円以下
    approvers: ["経理担当"]
    auto_approve: true      # 差異なしなら自動承認

  - max_amount: 1000000     # 100万円以下
    approvers: ["経理担当", "部門長"]
    auto_approve: false     # 必ず人間が承認

  - max_amount: null        # 100万円超
    approvers: ["経理担当", "部門長", "役員"]
    auto_approve: false

確認チェックリスト

  • 自動承認された請求書を定期的にサンプル抽出（月10%程度）して目視確認しているか
  • 「要確認」になった請求書の対応が支払期日前に完了しているか
  • インボイス番号エラーが出た取引先に連絡し、番号の再確認または修正請求書を依頼したか
  • AIの自動抽出で金額・支払期日が正しく読み取れているか（新規取引先の初回は必ず確認）
  • 承認済みのCSVを会計ソフトにインポートした後、合計金額が一致しているか確認したか
  • 承認フローの設定（金額閾値・承認者）が変更になった場合、config.yamlを更新したか
  • 月次の処理件数・エラー率をモニタリングしてAIの精度低下を早期に検知しているか

注意点

  • 初期精度は70〜80%程度。誤検出のパターンを学習して改善していく運用が必要。
  • 請求書のフォーマットは取引先によって異なる。フォーマットが特殊な取引先は個別対応ルールを追加する。
  • 完全自動承認は避ける。必ず人間がサンプルチェックを行い、自動化の精度を定期的に確認する。
