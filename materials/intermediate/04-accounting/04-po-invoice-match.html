発注書 vs 請求書：品目・数量・税額の差異を自動検出

★★★ 設定半日〜 3方向照合を自動化 OCR Claude Code Python
← 一覧に戻る

概要

発注書・納品書・請求書の3つの書類をPDFで渡すと、品目名・数量・単価・税額を自動で突合し、差異がある箇所をハイライトしたレポートを出力する。手作業での照合ミスをゼロにする。

必要な準備

Python（PyPDF2 または pdfplumber） Claude Code（PDFの読み取り能力を活用） Google Document AI（スキャンPDFの場合）

    3書類の突合ロジックを実装
    # matcher.py（Claude Codeで生成・抜粋） def match_documents(po: dict, receipt: dict, invoice: dict) -> list: discrepancies = [] for po_item in po['items']: inv_item = find_matching_item(po_item['name'], invoice['items']) if not inv_item: discrepancies.append({ "type": "品目不一致", "detail": f"発注書の'{po_item['name']}'が請求書にない" }) elif po_item['quantity'] != inv_item['quantity']: discrepancies.append({ "type": "数量不一致", "detail": f"'{po_item['name']}': 発注{po_item['quantity']} vs 請求{inv_item['quantity']}" }) return discrepancies
  • 4
    差異レポートを出力

    差異がある場合は「差異あり・要確認」として担当者にSlack/メールで通知。差異なしは自動承認キューに送る。

注意点

  • 品目名の表記揺れ（スペース・略称・漢字/カナの違い等）に対応するため、類似度マッチングを実装する。
  • スキャンPDFはOCRが必要。Google Document AIはGCP設定が必要だが精度が高い。
  • 差異が見つかった場合の確認フローを事前に決めておく（誰が・いつ・どのように確認するか）。
