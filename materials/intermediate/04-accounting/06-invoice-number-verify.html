適格請求書発行事業者番号を国税庁APIで一括検証

★★ 設定1〜2時間 インボイス番号を自動照合 国税庁公表システムAPI Claude Code Python
← 一覧に戻る

概要

取引先から受け取った請求書のインボイス番号（T+13桁の登録番号）を、国税庁の適格請求書発行事業者公表システムAPIに問い合わせて一括検証する。無効な番号の請求書を混入させないためのチェックを自動化する。

国税庁公表システムAPIについて

# APIエンドポイント（2025年時点） ベースURL: https://web-api.invoice-kohyo.nta.go.jp/1/invoice/ 認証：不要（無料で利用可能） 制限：1日あたり1,000件まで # 検索例 GET /1/invoice/T1234567890123 # レスポンス：事業者名・所在地・登録状況・登録日・取消日

設定手順

  • 1
    番号リストを準備する

    取引先管理シートまたは請求書フォルダから、確認すべきインボイス番号のリストを作成する。

    # invoice_numbers.csv company_name,invoice_number 株式会社〇〇,T1234567890123 〇〇合同会社,T9876543210987
  • 2
    一括検証スクリプトを作成（Claude Codeで生成）
    # verify_invoice_numbers.py import requests, pandas as pd, time def verify_number(invoice_number: str) -> dict: url = f"https://web-api.invoice-kohyo.nta.go.jp/1/invoice/{invoice_number}" try: response = requests.get(url, timeout=10) if response.status_code == 200: data = response.json() return { "status": "有効", "registered_name": data.get("registrant", {}).get("name"), "registration_date": data.get("registrant", {}).get("registrationDate"), "is_cancelled": data.get("registrant", {}).get("process") == "02" } else: return {"status": "番号なし（無効）"} except Exception as e: return {"status": f"確認失敗: {e}"} def batch_verify(csv_path: str): df = pd.read_csv(csv_path) results = [] for _, row in df.iterrows(): result = verify_number(row['invoice_number']) results.append({**row.to_dict(), **result}) time.sleep(0.5) # API制限対策 pd.DataFrame(results).to_csv("verification_result.csv", index=False) print("完了：verification_result.csvに出力しました")
  • 3
    請求書処理パイプラインに組み込む

    前のステップ（請求書確認作業の自動化）と組み合わせて、PDFからインボイス番号を抽出→自動検証→結果を確認チェックリストに追記する流れを構築する。

注意点

  • 国税庁APIは日次1,000件の制限がある。大量の請求書を処理する場合は複数日に分けるか、結果をキャッシュして再利用する。
  • 登録番号が有効でも、請求書の発行者名と国税庁DBの登録名が一致しない場合は要確認。
  • 取消・失効している番号（process=02）は仕入税額控除が認められないため、即座に取引先に確認する。
  • APIの仕様は変更される場合がある。定期的に国税庁の公表情報を確認する。
