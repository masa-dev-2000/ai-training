freee API連携 仕訳異常値検出・レポート出力

★★★ 設定半日〜 件数に関わらず処理時間一定 freee API Claude Code Python
← 一覧に戻る

概要

freee APIから当月の仕訳データを取得し、Claudeが統計的な異常値・計上ミスの疑いがある仕訳を自動検出してレポートを出力する。仕訳件数が増えても処理時間は一定に保たれる。

必要な準備

freee会計アカウント（スターター以上） freee APIアプリ登録（開発者向け） Python（requests, anthropic, pandas） Claude Code

設定手順

  • 1
    freee APIのOAuth認証を設定

    freee開発者サイト（developer.freee.co.jp）でアプリを作成し、アクセストークンを取得。スコープ：accounting

    FREEE_ACCESS_TOKEN=your_access_token FREEE_COMPANY_ID=1234567
  • 2
    仕訳データ取得スクリプトを作成
    # fetch_journals.py（Claude Codeで生成・抜粋） import requests, pandas as pd def get_journals(start_date: str, end_date: str) -> pd.DataFrame: url = "https://api.freee.co.jp/api/1/deals" headers = {"Authorization": f"Bearer {FREEE_ACCESS_TOKEN}"} params = { "company_id": FREEE_COMPANY_ID, "start_issue_date": start_date, "end_issue_date": end_date, "limit": 100 } all_deals = [] while True: response = requests.get(url, headers=headers, params=params).json() all_deals.extend(response.get("deals", [])) if not response.get("meta", {}).get("next_page"): break params["offset"] = params.get("offset", 0) + 100 return pd.DataFrame(all_deals)
  • 3
    異常検出ロジックをClaude Codeで設計
    プロンプト例
    freeeから取得した仕訳データ（journals_df）の異常値を検出するロジックを実装してください。

検出項目：
1. 同一勘定科目への極端に大きい/小さい金額（前月比300%超等）
2. 土日・祝日に計上された仕訳
3. 摘要が空白または「テスト」等の疑わしい仕訳
4. 借方貸方の不一致（合計金額が合わない）
5. 前月に同一取引先・同一金額がない突発的な支払い

異常と判定した仕訳の一覧とAIによる判定理由をMarkdownレポートで出力してください。
  • 4
    月次自動実行を設定

    毎月5日（前月分のデータが揃うタイミング）にcronで自動実行。担当者にレポートをメール送信する。

設定例（ケース別）

月次定期実行 — cron設定例
毎月5日の午前9時に前月分の仕訳データを取得・チェックし、
結果をPDFレポートとしてメール送信するcron設定を作成してください。

# crontab
0 9 5 * * /usr/bin/python3 /opt/journal_check/run_monthly.py

# run_monthly.py（Claude Codeで生成）
import datetime

last_month = datetime.date.today().replace(day=1) - datetime.timedelta(days=1)
start = last_month.replace(day=1).strftime("%Y-%m-%d")
end = last_month.strftime("%Y-%m-%d")

journals = get_journals(start, end)
anomalies = detect_anomalies(journals)
send_report_email(anomalies, to="keiri@company.co.jp")
手動トリガー実行 — 任意期間を指定して実行
特定の期間を指定してコマンドラインから即座に異常値チェックを実行する。

# コマンド例
python run_check.py --start 2026-04-01 --end 2026-04-30

# run_check.py（抜粋）
import argparse
parser = argparse.ArgumentParser()
parser.add_argument("--start", required=True)
parser.add_argument("--end", required=True)
args = parser.parse_args()

journals = get_journals(args.start, args.end)
anomalies = detect_anomalies(journals)

# 結果をターミナルに表示 + CSVに保存
print(f"検出件数: {len(anomalies)}件")
anomalies.to_csv("anomaly_report.csv", index=False, encoding="utf-8-sig")
特定勘定科目の監視 — 重点チェック設定
外注費・広告宣伝費など金額が大きくなりやすい科目に特化した
厳格な監視ルールを設定する。

# 監視対象科目と閾値設定（config.yaml）
watch_accounts:
  - name: 外注費
    threshold_ratio: 1.5   # 前月比150%超で警告
    max_single: 3000000    # 1件あたり上限額（300万円）
  - name: 広告宣伝費
    threshold_ratio: 2.0
    max_single: 5000000
  - name: 接待交際費
    threshold_ratio: 1.3
    max_single: 200000

# 閾値超えは即座にSlack通知
if anomaly["amount"] > config["max_single"]:
    slack_notify(f"警告: {anomaly['account']} に{anomaly['amount']:,}円の計上")

確認チェックリスト

  • 検出された異常仕訳を原票（領収書・請求書）と照合したか
  • 土日・祝日計上の仕訳に業務上の合理的な理由があるか確認したか
  • 借方貸方の不一致は丸め誤差か入力ミスかを特定したか
  • 前月比300%超の計上は経営者・上長への報告が必要か検討したか
  • 摘要が空白の仕訳は内容を特定し記載を補完したか
  • 修正仕訳が必要な場合、税理士に確認してからfreeeに反映したか
  • APIアクセストークンの有効期限が切れていないか（リフレッシュ設定の確認）

注意点

  • freee APIのアクセストークンは有効期限があるため、リフレッシュトークンを使った自動更新処理を実装する。
  • 異常検出はあくまでフラグ立て。実際の修正はfreee上で経理担当者が行う。
  • 会計データは高機密情報。APIアクセストークンは環境変数で管理し、コードに直書きしない。
