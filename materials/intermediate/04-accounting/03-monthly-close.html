月次決算チェック：前払費用の未計上・減価償却漏れ・未払社会保険料の計上漏れを自動指摘

★★★ 設定半日〜 月次決算チェックの自動化 会計ソフトAPI Claude Code Excel/Sheets
← 一覧に戻る

概要

会計ソフト（freee / マネーフォワード / 弥生等）から月次データを取得し、Claudeが前払費用の未計上・減価償却の漏れ・未払社会保険料の計上漏れを自動チェック。修正仕訳の候補まで提案する。

完成イメージ（出力例）

↓ 試算表CSV（2ヶ月分）を処理した場合のチェック結果レポート例

月次決算チェックレポート — 2026年4月末時点
チェック項目判定詳細・修正仕訳の候補
保険料（年払い）の月次按分 計上漏れ 損害保険料¥360,000を4月に一括計上。前払費用¥300,000（5〜12月分）の振替が未処理。
修正仕訳案：（借）前払費用 300,000 ／（貸）損害保険料 300,000
ソフトウェアライセンスの前払い処理 OK 月次按分が正しく計上されています（¥45,000／月）
減価償却費（前月比較） 要確認 前月比¥80,000増加。3月に固定資産取得の記録あり。台帳との照合を推奨。
社会保険料（未払計上） 計上漏れ 4月分社保（翌月納付）の未払計上なし。
修正仕訳案：（借）法定福利費 185,000 ／（貸）未払費用 185,000
インボイス番号未確認の仕入 要確認 4件（計¥320,000）のインボイス番号が未登録。仕入税額控除に影響する可能性あり。

※ 修正仕訳の候補は必ず税理士・経理担当者が承認した上で会計ソフトへ反映すること。

チェック項目一覧

# monthly_close_checks.md ## 前払費用関連 - 保険料（年払い）の月次按分が計上されているか - ソフトウェアライセンス費用の前払い処理が正しいか - 翌月以降分の費用が当月に一括計上されていないか ## 減価償却関連 - 前月比で減価償却費の変動がないか（新規取得・売却の反映漏れ） - 耐用年数が過ぎた資産が引き続き償却されていないか ## 未払・未収関連 - 社会保険料（翌月納付分）の未払計上があるか - 従業員給与の締め後から月末までの未払賃金 - 期末に多発する売上の計上漏れ（検収ベースと請求ベースの差） ## 消費税関連 - 消費税区分（課税/非課税/免税）の誤りがないか - インボイス番号が未確認の仕入れがないか

設定手順

  • 1
    前月・当月の試算表CSVをエクスポート

    会計ソフトから試算表（勘定科目別残高）を2ヶ月分CSVでエクスポートする。freeeの場合：レポート → 試算表 → CSV出力。

  • 2
    チェックスクリプトをClaude Codeで生成
    プロンプト例
    先月と今月の試算表CSV（prev_month.csv, this_month.csv）を比較し、
monthly_close_checks.mdの各チェック項目を自動検証するPythonスクリプトを作成してください。

出力：
- チェック結果一覧（OK/要確認/NG）
- 要確認・NGの場合は具体的な勘定科目名・金額・修正仕訳の候補
- Markdownレポートとして出力
  • 3
    固定資産台帳との照合を追加

    固定資産台帳Excelを読み込み、減価償却費の計算値と会計帳簿の計上額を自動照合する機能を追加する。

  • 4
    月次チェックを定型業務として自動化

    毎月5日にCSVエクスポート → 自動チェック → レポートをメール送信という流れをcronで自動化する。

設定例（ケース別）

月末チェック実行 — 毎月5日に自動実行するスクリプト
先月と今月の試算表CSVを読み込んで月次チェックを実行し、
結果を担当者のメールアドレスに送信するスクリプトを作ってください。

# 実行方法
python monthly_close.py --month 2026-04

# 出力ファイル
- monthly_report_2026-04.md  ← Markdownレポート
- monthly_report_2026-04.csv ← チェック項目一覧（判定・詳細）

# crontab（毎月5日 9:00実行）
0 9 5 * * python /opt/monthly_close/monthly_close.py --auto
四半期決算チェック — 追加チェック項目付き
四半期末（3・6・9・12月）には通常の月次チェックに加えて
以下を追加実行するよう条件分岐を設定してください。

追加チェック項目（四半期のみ）：
- 棚卸資産の評価損（前期末比で急減していないか）
- 引当金の積み増し漏れ（貸倒引当金・賞与引当金）
- 繰延税金資産・負債の見直し必要性
- 関係会社との取引（内部消去もれ）

# config.yaml に四半期フラグを設定
check_mode: quarterly  # monthly / quarterly / annual
カスタムチェック項目追加 — 会社固有のルールを登録
業種・会社規模に応じたカスタムチェック項目を追加する方法。

# custom_checks.yaml に追記
custom_rules:
  - name: "建設仮勘定の振替確認"
    account: "建設仮勘定"
    condition: "前月から増減あり → 固定資産への振替が完了しているか確認"
    alert_threshold: 1000000  # 100万円超で警告

  - name: "役員報酬の定期同額確認"
    account: "役員報酬"
    condition: "前月比で差異がある場合は税務上の問題を警告"
    tolerance: 0  # 差異ゼロが必須

# Claude Codeにカスタムルールを渡してチェックロジックを自動生成
python generate_custom_checks.py --config custom_checks.yaml

確認チェックリスト

  • 前払費用の月次按分が正しい金額・期間で計算されているか（計算式を確認）
  • 固定資産台帳の減価償却計算が法定耐用年数に基づいているか
  • 社会保険料の未払計上額が実際の給与台帳と一致しているか
  • インボイス番号未確認の仕入は仕入税額控除を保留にする対応をとったか
  • AIが提案した修正仕訳を、必ず税理士または経理責任者が確認したか
  • 修正仕訳を会計ソフトに反映後、試算表を再出力して合計が合っているか確認したか
  • チェック結果レポートを指定フォルダに保存・共有したか（バックアップ含む）

注意点

  • AIのチェック結果は一次スクリーニング。最終的な修正判断は税理士・経理担当者が行う。
  • 業種・会社規模によってチェック項目は異なる。初回は税理士と一緒にチェックリストをカスタマイズする。
  • 会計データは高機密。ファイルのアクセス権限と保存場所を適切に管理する。
