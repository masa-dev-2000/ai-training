法務プレイブックと過去契約書を参照し、条項ごとに差異・リスクを瞬時に判定

★★★★ エンジニア向け 構築に数日〜 ベクトルDB RAG Claude Code
← 一覧に戻る

概要

自社の法務プレイブック（許容条件の基準書）と過去の締結済み契約書をベクトルデータベースに格納し、新規契約書の各条項を照合。プレイブックとの差異・損害賠償上限なし・管轄問題等のリスクをリアルタイムで判定するRAGシステムを構築する。

■ 第8条（損害賠償）　リスク：高 🔴
【契約書の記載】
「受託者は、本業務の履行に関して委託者に生じた一切の損害を
 賠償する責任を負うものとする。」

【プレイブック基準】
損害賠償は「故意または重過失の場合に限り」「本契約に基づく
報酬総額を上限として」補償する旨を定めること。

【差異】
・免責条件（故意/重過失限定）の記載なし
・上限額の規定なし → 無制限の賠償リスク

【修正案】
「受託者が本業務の履行に関して故意または重過失により委託者に
 損害を与えた場合、その賠償額は本契約における報酬総額を上限
 とする。間接損害・逸失利益は除く。」

━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 第12条（知的財産権の帰属）　リスク：中 🟡
【契約書の記載】
「本業務の成果物に関する著作権は、受託者に帰属する。」

【プレイブック基準】
委託業務の成果物の著作権は委託者に帰属、または
著作権譲渡を明記することを原則とする。

【差異】
成果物の著作権が受託者（外部）に残る設計になっている。

【修正案】
「本業務の成果物に関する著作権（著作権法第27条・28条の権利
 を含む）は、委託者に帰属するものとする。」または
「受託者は委託者に対し著作権を譲渡する」旨を追記。

━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 第15条（競業避止）　リスク：中 🟡
【契約書の記載】
「受託者は、契約終了後2年間、競合他社への役務提供を行わない。」

【プレイブック基準】
競業避止は「合理的な範囲（期間・地域・業務内容を限定）」に
とどめること。過度な制限は無効となる可能性がある。

【差異】
「競合他社」の定義が不明確。地域的限定もなし。
有効性に疑義あり、かつ受託者にとっても過剰負担の可能性。

【修正案】
「競合他社」の具体的定義を列挙、または「〇〇分野において
 直接競合する」等の限定を設ける。

必要な準備

Python（langchain / llama-index） ベクトルDB（Chroma / Pinecone / Qdrant） 法務プレイブック（PDF / Word） 過去契約書データ（匿名化済み） Anthropic APIキー

設定手順

  • 1
    法務プレイブックをベクトルDBに格納
    pip install langchain chromadb anthropic pypdf2 # ingest_playbook.py（Claude Codeで生成） from langchain.document_loaders import PyPDFLoader from langchain.text_splitter import RecursiveCharacterTextSplitter from langchain_community.vectorstores import Chroma from langchain_anthropic import AnthropicEmbeddings loader = PyPDFLoader("legal_playbook.pdf") docs = loader.load() splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50) chunks = splitter.split_documents(docs) vectordb = Chroma.from_documents(chunks, AnthropicEmbeddings(), persist_directory="./legal_db")
  • 2
    契約書の条項を分割して格納

    過去契約書も同様にチャンク化してベクトルDB（別コレクション）に格納。各チャンクに「契約種別・リスクレベル・特記事項」のメタデータを付与する。

  • 3
    条項審査エンジンを実装
    # contract_reviewer.py（Claude Codeで生成・抜粋） def review_clause(clause_text: str) -> dict: # プレイブックから関連基準を検索 relevant_standards = vectordb.similarity_search(clause_text, k=3) # Claudeで差異分析 response = client.messages.create( model="claude-opus-4-7", messages=[{"role": "user", "content": f"以下の契約条項を審査してください。\n\n" f"条項：{clause_text}\n\n" f"自社基準：{[d.page_content for d in relevant_standards]}\n\n" f"差異・リスク（損害賠償上限・管轄・秘密保持期間等）を指摘し、" f"修正案を提示してください。"}] ) return {"risk_level": extract_risk(response), "details": response.content[0].text}
  • 4
    審査レポートを生成する

    全条項を順番に審査し、リスクレベル別の一覧表と修正提案付きレポートをMarkdown/PDFで出力する。

設定例（ケース別）

新規取引先の契約 — 基本の審査プロセス
新規取引先から受領した契約書を審査してください。

【契約の概要】
取引先：〇〇株式会社（国内）
契約種別：業務委託契約
取引内容：Webシステム開発の委託

【審査方針】
1. 自社法務プレイブックとの差異を全条項で確認する
2. 過去の類似契約（同業種・同規模）と比較する
3. リスクの高い順に優先して修正交渉する
4. 取引関係の維持を考慮し、最低限の修正点に絞る

【重点確認事項】
- 損害賠償の上限と免責条件
- 知的財産権の帰属（成果物・既存ノウハウ）
- 機密保持の範囲と期間
- 解除条件と違約金

各条項のリスク評価と優先度（交渉必須/推奨/許容）を
表形式で出力してください。
既存契約の更新 — 変更箇所の差異確認に特化
既存契約の更新版（相手方ドラフト）を審査してください。

【前回契約（締結済み）の主要条件】
- 損害賠償上限：報酬総額（〇〇円）
- 秘密保持期間：契約終了後2年
- 管轄：東京地方裁判所
- 知的財産：成果物は委託者帰属

【今回の更新版で変更されている可能性がある箇所】
（ここに更新版の全文または差分をペースト）

確認してほしいこと：
1. 前回から変更された条項の特定
2. 変更により自社に不利になった点の指摘
3. 変更を受け入れてよいか・再交渉すべきかの判断
4. 更新に際して法令改正等による修正が必要な点
海外企業との契約 — 国際契約特有のリスクに注目
海外企業との契約書を審査してください。

【契約の概要】
相手方：〇〇 Inc.（米国・デラウェア州法人）
契約種別：ソフトウェアライセンス契約（英文契約）
準拠法：米国カリフォルニア州法（相手方案）

【審査で特に確認してほしい点】
1. 準拠法・管轄（日本法・日本の裁判所への変更交渉余地）
2. 限定保証・免責事項（AS-ISが多い）の範囲
3. 間接損害・逸失利益の免責条項の有無
4. GDPR・個人情報保護法（日本）双方への対応状況
5. 仲裁条項（国際仲裁機関の指定は妥当か）
6. クラスアクション放棄条項など日本法上問題になる条項

【契約書本文】
（ここに英文または日本語訳をペースト）

日本企業として受け入れ可能な条件か、
修正交渉すべき点を具体的に指摘してください。

確認チェックリスト

  • 法務プレイブックが最新の社内方針・法令に合わせて更新されているか
  • ベクトルDBに格納した過去契約書の個人情報が匿名化されているか
  • 審査対象の契約書をAPIに送信することが社内のデータ取り扱い規定上問題ないか
  • AIが「問題なし」と判定した条項についても弁護士が目視確認したか
  • リスク「高」の条項について、交渉しない場合のリスクを経営層に共有したか
  • 外国法が準拠法の場合、現地の専門弁護士に確認を取ったか
  • 審査レポートの保存先と保管期間が定められているか

注意点

  • プレイブックは最新の法律・社内方針に合わせて定期的にアップデートが必要。古い基準で審査し続けると誤判定が増える。
  • ベクトルDBのchunk_sizeは条項の長さに合わせて調整が必要。条項の途中で切れると判断精度が下がる。
  • 個人情報が含まれる過去契約書は格納前に匿名化処理を施すこと。
