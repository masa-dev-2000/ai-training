法務プレイブックと過去契約書を参照し、条項ごとに差異・リスクを瞬時に判定

★★★★ エンジニア向け 構築に数日〜 ベクトルDB RAG Claude Code
← 一覧に戻る

概要

自社の法務プレイブック（許容条件の基準書）と過去の締結済み契約書をベクトルデータベースに格納し、新規契約書の各条項を照合。プレイブックとの差異・損害賠償上限なし・管轄問題等のリスクをリアルタイムで判定するRAGシステムを構築する。

注意

このシステムはエンジニアリングスキルが必要です。法的判断の最終確認は必ず弁護士が行ってください。

必要な準備

Python（langchain / llama-index） ベクトルDB（Chroma / Pinecone / Qdrant） 法務プレイブック（PDF / Word） 過去契約書データ（匿名化済み） Anthropic APIキー

設定手順

  • 1
    法務プレイブックをベクトルDBに格納
    pip install langchain chromadb anthropic pypdf2 # ingest_playbook.py（Claude Codeで生成） from langchain.document_loaders import PyPDFLoader from langchain.text_splitter import RecursiveCharacterTextSplitter from langchain_community.vectorstores import Chroma from langchain_anthropic import AnthropicEmbeddings loader = PyPDFLoader("legal_playbook.pdf") docs = loader.load() splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50) chunks = splitter.split_documents(docs) vectordb = Chroma.from_documents(chunks, AnthropicEmbeddings(), persist_directory="./legal_db")
  • 2
    契約書の条項を分割して格納

    過去契約書も同様にチャンク化してベクトルDB（別コレクション）に格納。各チャンクに「契約種別・リスクレベル・特記事項」のメタデータを付与する。

  • 3
    条項審査エンジンを実装
    # contract_reviewer.py（Claude Codeで生成・抜粋） def review_clause(clause_text: str) -> dict: # プレイブックから関連基準を検索 relevant_standards = vectordb.similarity_search(clause_text, k=3) # Claudeで差異分析 response = client.messages.create( model="claude-opus-4-7", messages=[{"role": "user", "content": f"以下の契約条項を審査してください。\n\n" f"条項：{clause_text}\n\n" f"自社基準：{[d.page_content for d in relevant_standards]}\n\n" f"差異・リスク（損害賠償上限・管轄・秘密保持期間等）を指摘し、" f"修正案を提示してください。"}] ) return {"risk_level": extract_risk(response), "details": response.content[0].text}
  • 4
    審査レポートを生成する

    全条項を順番に審査し、リスクレベル別の一覧表と修正提案付きレポートをMarkdown/PDFで出力する。

注意点

  • プレイブックは最新の法律・社内方針に合わせて定期的にアップデートが必要。古い基準で審査し続けると誤判定が増える。
  • ベクトルDBのchunk_sizeは条項の長さに合わせて調整が必要。条項の途中で切れると判断精度が下がる。
  • 個人情報が含まれる過去契約書は格納前に匿名化処理を施すこと。
