NDAをSlackにアップロードするだけでリスク評価・フォルダ格納・ファイルリネームまで自動化

★★★ 設定半日〜 NDA処理を全自動化 Slack MCP Google Drive Claude Code

概要

#法務チャンネルにNDAのPDFをアップロードするだけで、AIがテキストを抽出・リスク評価・問題条項の指摘を行い、結果をSlackに返信。同時にGoogle DriveへのNDA分類・フォルダ格納・ファイルリネームまで自動処理する。

必要な準備

Slack Bot（files:read、chat:write スコープ） Google Drive API（サービスアカウント） Python（PyPDF2、anthropic、slack-bolt） Claude Code

設定手順

  • 1
    Slack Botを作成してイベント購読を設定

    api.slack.com → Event Subscriptions → message.files イベントを購読。#法務チャンネルのファイルアップロードを検知する。

  • 2
    PDFテキスト抽出 + リスク評価スクリプトを生成
    Claude Codeへのプロンプト
    NDA自動処理システムを作成してください。

処理フロー：
1. SlackからPDFファイルを受け取る（slack-boltを使用）
2. PyPDF2でテキストを抽出
3. Claudeでリスク評価を実施
   - 秘密保持期間（短すぎないか）
   - 損害賠償の上限設定の有無
   - 管轄裁判所（相手方有利でないか）
   - 目的外使用の制限
   - 残存条項（契約終了後の義務）
4. リスクレベル（高/中/低）と具体的な問題点をSlackに返信
5. Google DriveのNDA管理フォルダに保存
   ファイル名：YYYYMMDD_相手方企業名_NDA.pdf
  • 3
    リスク評価プロンプトを定義する
    # nda_review_prompt.txt 以下のNDA文書を法務的観点から審査してください。 評価項目： 1. 秘密情報の定義（広すぎないか・曖昧でないか） 2. 秘密保持期間（契約終了後何年か） 3. 損害賠償条項（上限額の設定・懲罰的賠償の有無） 4. 目的の限定（明確に記載されているか） 5. 管轄・準拠法（相手方有利でないか） 6. 残存条項 各項目をリスクレベル（高/中/低）で評価し、 問題がある場合は具体的な条項番号と修正提案を記載してください。
  • 4
    スクリプトをサーバーにデプロイして常駐化

    VPSまたはCloud Run等にデプロイして24時間稼働させる。Slackへのアップロードを待ち受ける。

設定例（ケース別）

NDA受領時 — 相手方から受け取ったNDAのリスク評価
以下のNDA文書（相手方ドラフト）を審査してください。

【評価観点】
- 自社に不利な条項がないか
- 秘密保持期間は自社標準（3年）以上か
- 損害賠償上限の記載があるか
- 管轄裁判所は自社所在地または東京か
- 目的外使用の制限が明確か

【NDA本文】
（ここにテキスト抽出したNDA内容をペースト）

各項目をリスクレベル（高/中/低）で評価し、
修正交渉すべき条項と具体的な修正案を提示してください。
NDAドラフト依頼時 — 自社から提示するNDAを作成
新規取引先向けのNDAドラフトを作成してください。

【取引概要】
相手方：〇〇株式会社
取引内容：システム開発の打ち合わせ・仕様検討
開示方向：相互開示（双方向）
想定期間：6か月

【自社の標準条件】
- 秘密保持期間：契約終了後3年
- 損害賠償上限：過去12か月の取引総額
- 管轄：東京地方裁判所
- 準拠法：日本法

以上を踏まえ、自社に有利で、かつ相手方も受け入れやすい
バランスの取れたNDAドラフトを作成してください。
修正依頼時 — 相手方から修正案が来た場合の差異確認

変更箇所とリスクを確認してください。

【自社ドラフト（主要条項）】
第〇条（秘密保持期間）：契約終了後3年
第〇条（損害賠償）：過去12か月の取引総額を上限
第〇条（管轄）：東京地方裁判所

【相手方修正案】
（ここに相手方の修正テキストをペースト）

確認事項：
1. 変更された条項の一覧
2. 各変更のリスク評価（高/中/低）
3. 受け入れ可能か・再交渉が必要かの判断
4. 代替案の提示（再交渉が必要な場合）

確認チェックリスト

  • AIのリスク評価結果を法務担当者・顧問弁護士が確認したか
  • リスク「高」と判定された条項の対応方針を決定したか
  • 相手方との管轄・準拠法の合意が取れているか（国内法か）
  • 損害賠償上限が記載されており、自社に過剰な責任が生じないか
  • 秘密情報の定義に口頭開示が含まれる場合、確認の手順を定めているか
  • NDAに個人情報の取り扱いに関する記載が必要かを確認したか
  • Google Driveへの格納先フォルダが正しいか（「締結済」と「検討中」を区別）
  • ファイル名（YYYYMMDD_企業名_NDA.pdf）が正しく付与されているか

注意点

  • NDA文書には機密情報が含まれる。使用するClaude APIがデータ学習に使われない設定（API経由の利用）であることを確認する。
  • スキャンPDFはOCR処理が必要。pytesseractを追加するか、Adobe PDF APIを使うとテキスト精度が上がる。
  • AIの評価結果はあくまで一次スクリーニング。リスク「高」判定のNDAは必ず法務担当者・顧問弁護士が確認する運用フローを設定すること。
