契約更新期限・自動更新条項 横断管理 ＋ アラート通知

★★ 設定1〜2時間 更新見落としゼロへ Google Sheets Claude Code Slack / メール
← 一覧に戻る

概要

全契約書の更新期限・自動更新条項・解約通知期限をGoogle Sheetsで一元管理し、期限30日前・7日前・当日にSlackとメールで自動アラートを送信する。担当者が変わっても更新見落としが起きない体制を構築できる。

完成イメージ（出力例）

↓ 毎朝9時に自動実行され、期限が近い契約の担当者宛に以下のアラートメールが送信される

設定手順

  • 1
    契約管理スプレッドシートを設計する
    # 列構成（Google Sheets） A: 契約ID B: 相手方企業名 C: 契約種別（NDA/業務委託/保守等） D: 契約開始日 E: 契約終了日 F: 自動更新（あり/なし） G: 解約通知期限（終了〇日前まで） H: 担当者名 I: 担当者メール J: 備考 K: ステータス（有効/更新済/終了）
  • 2
    既存契約書からデータを一括取り込み（Claude Codeで自動化）

    契約書PDFが多数ある場合、Claude Codeに「これらのPDFから契約管理シートの各列に必要な情報を抽出してCSVで出力して」と指示すると大量のデータ入力を自動化できる。

  • 3
    アラートスクリプトを作成（Claude Codeで生成）
    # contract_alert.py（Claude Codeで生成・抜粋） import gspread, anthropic, smtplib, json from datetime import datetime, timedelta def check_and_alert(): ws = gc.open("契約管理").sheet1 records = ws.get_all_records() today = datetime.today().date() for row in records: if row['ステータス'] != '有効': continue end_date = datetime.strptime(row['契約終了日'], '%Y/%m/%d').date() days_left = (end_date - today).days if days_left in [30, 7, 0]: message = f"【契約更新アラート】{row['相手方企業名']}\n" \ f"契約終了日：{row['契約終了日']}（あと{days_left}日）\n" \ f"自動更新：{row['自動更新']}\n担当：{row['担当者名']}" send_slack(message) send_email(row['担当者メール'], message)
  • 4
    毎朝9時に自動実行
    # crontab 0 9 * * * python contract_alert.py
  • 5
    AIによる解約通知期限の計算（任意）

    「解約通知期限は終了60日前まで」のような条件をClaudeが自動計算し、「今週中に解約通知を送るべき契約一覧」を毎週月曜に生成する機能を追加できる。

【変更後のアラートスケジュール】
- 90日前：担当者へ事前通知（更新か終了かを決める期限）
- 60日前：担当者＋上長へ通知（方針決定の催促）

- 自動更新ありの契約は「解約通知期限」（G列）に基づいて
  別途アラートを出す
- 解約通知期限が7日以内に迫った場合は緊急フラグを付ける

contract_alert.py を上記仕様に書き直してください。
特定カテゴリの抽出 — 契約種別・担当者別のフィルタリング
月次の法務レポート用に、契約管理シートから
特定条件でデータを抽出するスクリプトを作成してください。

【抽出条件A：今月中に期限が来る契約】
- ステータス：有効
- 契約終了日：今月末日まで
- 出力：企業名・契約種別・終了日・自動更新・担当者

【抽出条件B：自動更新条項あり・解約通知期限が60日以内】
- ステータス：有効
- 自動更新：あり
- 解約通知期限（G列の数値）× 日前が60日以内に相当する契約
- 出力：企業名・終了日・解約通知送付期限・担当者

【抽出条件C：担当者別の契約件数・期限一覧】
- 担当者名でグループ化
- 各担当者の有効契約数・最も近い期限を一覧表示

各抽出結果をSlackの#法務チャンネルに毎月1日に投稿する
機能も合わせて実装してください。

確認チェックリスト

  • 自動更新条項のある契約は解約通知期限（G列）が正しく入力されているか
  • アラートメールの送信元アドレスが法務担当者に認識されるアドレスになっているか
  • 担当者のメールアドレスが最新の状態に保たれているか（退職・異動に対応）
  • スプレッドシートの編集権限が適切に設定されているか（誰でも削除できない状態）
  • アラートを受け取った担当者が次のアクションを実施したかを追跡できているか
  • 解約通知を送付する場合、書面・内容証明の要否を法務担当者に確認したか
  • テスト実行で実際にSlack・メールが正しく届くことを確認したか

注意点

  • 自動更新条項がある契約は解約通知期限（例：終了60日前）を必ずG列に記載する。この期限を逃すと不要な契約が自動更新されてしまう。
  • スプレッドシートは複数人で管理するため、編集権限と変更履歴を適切に設定する。
  • メール通知のSMTP設定にはGoogleアカウントのアプリパスワード（2段階認証が必要）を使用する。
