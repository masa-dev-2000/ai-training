Slack自動返信ボット：FAQ一次対応 → 未解決なら人間にエスカレーション

★★★ 設定半日〜 24時間FAQ対応を自動化 Slack API Claude Code FAQ DB
← 一覧に戻る

概要

Slackの特定チャンネル（サポートチャンネル等）に投稿されたメッセージをClaudeが分析し、FAQに基づいて一次回答を自動返信する。FAQ外の質問や複雑な内容は担当者にメンションして人間対応に切り替える。

システム構成

# 処理フロー ユーザー投稿 ↓ Slack Event API（message イベント） ↓ Claudeが質問を分析 ├── FAQで解決可能 → 自動返信 + FAQ URLリンク ├── FAQ外だが回答可能 → 返信 + "より詳しくは担当者まで" └── 複雑・クレーム・個人情報絡み → 担当者にメンション通知

設定手順

  • 1
    FAQデータベースを準備する

    MarkdownまたはJSONでFAQを作成。Q&Aの形式でまとめ、カテゴリ別に整理しておく。

    # faq.md（抜粋） ## 料金・支払い Q: 無料トライアルはありますか？ A: はい、14日間の無料トライアルをご用意しています。クレジットカード不要でご利用いただけます。 Q: 途中でプランを変更できますか？ A: いつでも変更可能です。アップグレードは即日、ダウングレードは次回更新日から適用されます。
  • 2
    Slack Botを作成してEvent APIを設定

    api.slack.com → アプリ作成 → Event Subscriptions → message.channels を購読。エンドポイントURLを設定する。

  • 3
    ボットサーバーを実装（Claude Codeで生成）
    # slack_bot.py（抜粋） from slack_bolt import App import anthropic app = App(token=SLACK_BOT_TOKEN, signing_secret=SLACK_SIGNING_SECRET) faq_content = open("faq.md").read() @app.event("message") def handle_message(event, say): if event.get("bot_id"): # ボット自身の投稿は無視 return user_message = event.get("text", "") response = claude_respond(user_message, faq_content) if response["needs_escalation"]: say(f"<@担当者のSlack ID> {response['summary']}") else: say(response["answer"])
  • 4
    エスカレーション条件を定義する

    「怒り・不満の感情が強い」「個人情報を含む」「FAQ外かつ回答に不確実性がある」「返金・解約要求」等をエスカレーション条件として設定する。

注意点

  • FAQは月1回以上更新する。古いFAQに基づいた誤回答がユーザーの信頼を損なう。
  • ボットが自動返信することをユーザーに明示する（「AIアシスタントが一次対応します」等）。
  • Slack Event APIのサーバーは24時間稼働が必要。VPS・Lambda等に適切にデプロイする。
  • エスカレーションが多い質問はFAQに追加していくことで、時間とともに自動解決率が上がる。
