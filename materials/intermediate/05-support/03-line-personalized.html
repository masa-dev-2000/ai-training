LINE MCPでユーザーごとにパーソナライズしたメッセージ自動送信

★★★★ エンジニア向け 構築数日〜 LINE Messaging API MCP ユーザーDB
← 一覧に戻る

概要

LINE Messaging APIをMCP経由でClaude Codeから操作できるようにし、ユーザーデータベース（購買履歴・属性・行動ログ）に基づいてパーソナライズされたメッセージを自動生成・送信するシステムを構築する。

必要な準備

LINE Developersアカウント LINE Messaging APIチャンネル（チャンネルアクセストークン） ユーザーデータベース（購買履歴・属性等） Python（line-bot-sdk） Claude Code

設定手順

  • 1
    LINE Messaging APIチャンネルを作成

    LINE Developers → プロバイダー作成 → Messaging APIチャンネル作成。チャンネルアクセストークン（長期）を発行して保存する。

  • 2
    LINE送信MCPサーバーを実装
    # mcp-line-server.py（抜粋） from mcp.server import Server from linebot.v3.messaging import ApiClient, Configuration, MessagingApi, TextMessage, PushMessageRequest app = Server("line-messaging") @app.tool() async def send_personalized_message(user_id: str, message: str): """LINEユーザーにメッセージを送信する""" config = Configuration(access_token=LINE_CHANNEL_ACCESS_TOKEN) with ApiClient(config) as api_client: api = MessagingApi(api_client) api.push_message(PushMessageRequest( to=user_id, messages=[TextMessage(type="text", text=message)] )) return f"送信完了: {user_id}"
  • 3
    ユーザーデータと組み合わせたメッセージ生成
    # パーソナライズロジック例 for user in users: profile = get_user_profile(user['line_id']) message = claude.generate(f""" 以下のユーザープロファイルに合わせた セグメント配信メッセージを生成してください： 名前：{profile['name']} 最終購入：{profile['last_purchase']} 購入カテゴリ：{profile['categories']} 休眠日数：{profile['days_since_purchase']} 目標：再購入促進 """) send_line_message(user['line_id'], message)
  • 4
    送信ログと効果測定の設定

    送信したメッセージ・開封率・クリック率をDBに記録し、Claude Codeで効果分析レポートを定期生成する。

注意点

  • LINE公式アカウントのメッセージ送信はプラン制限がある。大量配信はコスト計算を事前に確認する。
  • 個人に特化したメッセージ生成は個人情報保護法の対象。プライバシーポリシーを更新し、ユーザーの同意を取得すること。
  • 配信停止（ブロック）ユーザーへの送信はAPIレベルでエラーになる。エラーハンドリングとブロックリスト管理を実装する。
  • スパムと認識されるメッセージ頻度（週1〜2回が上限目安）に注意する。
