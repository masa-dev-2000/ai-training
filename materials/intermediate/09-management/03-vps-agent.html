VPS常駐エージェント：LINE WORKSから指示 → Gmail・Calendar・Drive・Notion横断操作

★★★★ エンジニア向け スマホから全ツールを一元操作 VPS LINE WORKS Claude API MCP
← 一覧に戻る

概要

VPS（仮想プライベートサーバー）にClaudeエージェントを常駐させ、LINE WORKSから自然言語で指示を送るとGmail・Googleカレンダー・Googleドライブ・Notionを横断して操作する。スマートフォンだけで全ツールの操作が完結する。

完成イメージ（出力例）

↓ LINE WORKSにメッセージを送信した場合のエージェントの応答例

LINE WORKS への送信メッセージ
「明日14時に鈴木さんとのミーティングをカレンダーに追加して。タイトルは"Q2振り返り"で90分。場所はオンライン（Google Meet）。」
エージェントの応答（LINE WORKS に自動返信）
承りました。以下の内容でGoogleカレンダーに登録します。確認してください。

タイトル：Q2振り返り
日時：2026年5月16日（木）14:00〜15:30（90分）
場所：Google Meet（自動生成リンクを添付）

登録してよろしいですか？（はい / いいえ）
「はい」と返信後のエージェントの応答
Googleカレンダーに登録しました。鈴木さんに招待メールを送信しました。Meet リンク：https://meet.google.com/xxx-yyyy-zzz

設定手順

  • 1
    VPSにサーバー環境を構築する

    さくらVPS・ConoHa等の月額500〜1,000円程度のVPSを用意する。Ubuntu 22.04 LTS推奨。

    # Python環境とMCPサーバーのインストール sudo apt update && sudo apt install -y python3-pip nodejs npm pip3 install anthropic flask gunicorn # Google Workspace MCPのインストール npm install -g @modelcontextprotocol/server-gdrive npm install -g @modelcontextprotocol/server-gcal # Notion MCPのインストール npm install -g @notionhq/notion-mcp-server
  • 2
    LINE WORKS Botを作成してWebhookを設定する

    LINE WORKS Developer Consoleでボットを作成し、Webhook URLにVPSのエンドポイントを設定する。

    # LINE WORKS Bot設定（Developer Console） Bot名：業務アシスタント Webhook URL：https://your-vps-ip/webhook 権限：メッセージ送受信 # 環境変数ファイル（.env） LINEWORKS_BOT_SECRET=xxx LINEWORKS_ACCESS_TOKEN=xxx LINEWORKS_CHANNEL_ID=xxx ANTHROPIC_API_KEY=sk-ant-xxx GOOGLE_CREDENTIALS_PATH=/app/credentials/google-creds.json NOTION_API_KEY=secret_xxx
  • 3
    Webhookサーバーを実装する
    # agent_server.py（抜粋） from flask import Flask, request, jsonify import anthropic, json, subprocess app = Flask(__name__) client = anthropic.Anthropic() MCP_CONFIG = { "mcpServers": { "gdrive": {"command": "npx", "args": ["-y", "@modelcontextprotocol/server-gdrive"]}, "gcal": {"command": "npx", "args": ["-y", "@modelcontextprotocol/server-gcal"]}, "notion": {"command": "npx", "args": ["-y", "@notionhq/notion-mcp-server"], "env": {"NOTION_API_KEY": os.environ["NOTION_API_KEY"]}} } } @app.route('/webhook', methods=['POST']) def webhook(): data = request.json user_message = data.get('content', {}).get('text', '') channel_id = data.get('channelId') # Claudeエージェントに処理を委任 result = run_agent(user_message) # LINE WORKSに返信を送信 send_lineworks_message(channel_id, result) return jsonify({"status": "ok"}) def run_agent(instruction: str) -> str: # Claude CLIをヘッドレスモードで実行（MCPが自動接続） proc = subprocess.run( ["claude", "--headless", "--print", "--mcp-config", json.dumps(MCP_CONFIG), instruction], capture_output=True, text=True, timeout=120 ) return proc.stdout.strip()
  • 4
    systemdで常駐サービスとして登録する
    # /etc/systemd/system/agent.service [Unit] Description=Business Agent Server After=network.target [Service] User=ubuntu WorkingDirectory=/app ExecStart=/usr/bin/gunicorn agent_server:app -b 0.0.0.0:5000 Restart=always EnvironmentFile=/app/.env [Install] WantedBy=multi-user.target
    # サービスを起動・自動起動設定 sudo systemctl daemon-reload sudo systemctl enable agent sudo systemctl start agent

確認チェックリスト

  • LINE WORKSの署名検証（Bot Secret による HMAC 検証）が実装されているか
  • VPSのSSHアクセスが鍵認証のみに制限されているか（パスワード認証を無効化）
  • Google OAuth2・Notion APIキー等の認証情報ファイルのパーミッションが600になっているか
  • 破壊的操作（Gmail送信・カレンダー作成・Notion書き込み）の前に確認メッセージを送る仕組みがあるか
  • エージェントのタイムアウト（120秒等）とトークン上限が設定されているか
  • LINE WORKSのボットAPIの利用プランとメッセージ送信上限を確認したか
  • systemdサービスが自動起動設定になっているか（VPS再起動後も動作するか）
  • Claude APIの利用コストを定期的に確認する仕組みがあるか

注意点

  • VPSはインターネット上に公開されるため、Webhookエンドポイントの認証（LINE WORKSの署名検証）を必ず実装する。署名未検証のまま公開すると第三者から任意の指示を実行される危険がある。
  • Googleの認証情報（OAuth2）はVPS上のファイルに保存される。VPSのSSHアクセスを鍵認証のみに制限し、認証情報ファイルのパーミッションを600に設定する。
  • エージェントはGmail送信・カレンダー作成・Notion書き込み等の破壊的操作も実行できる。重要な操作の前に確認メッセージを送り、「はい」の返信を受けてから実行する確認フローを設けることを強く推奨する。
  • LINE WORKSのAPIはエンタープライズプランが必要な機能がある。ボットAPIの利用可否・メッセージ送信上限を事前に確認する。
  • Claude APIの利用料金はメッセージ量に比例する。エージェントがループに入ったり大量の文書を処理するとコストが急増するため、タイムアウト（120秒等）とトークン上限を設定する。
