概要

CEO・CFO・CMO・CHO・CTO・COO の6役割に対応したカスタムスキルをClaude Codeに登録し、各部門の定型業務をコマンド1発で処理する体制を構築する。スキルごとに役割・ガイドライン・テンプレートを持たせることで、1人でも6部門の業務クオリティを維持できる。

部門別スキル一覧（例）

部門スキル例用途
CEO（経営）/board-report /okr-review /investor-update取締役会資料・OKR進捗・投資家向け報告
CFO（財務）/monthly-pl /cash-forecast /budget-variance月次PL分析・キャッシュ予測・予実差異レポート
CMO（マーケ）/campaign-brief /content-calendar /ad-report施策ブリーフ・コンテンツ計画・広告レポート
CHO（人事）/jd-draft /eval-summary /onboarding-plan求人票・評価まとめ・入社オンボーディング
CTO（技術）/pr-review /arch-review /incident-reportPRレビュー・設計評価・インシデントレポート
COO（オペレーション）/sop-draft /kpi-dashboard /vendor-eval業務手順書・KPI整理・ベンダー評価

設定手順

  • 1
    フォルダ構成を用意する
    # スキルと業務コンテキストのフォルダ構成 .claude/ skills/ ceo/ board-report.md okr-review.md investor-update.md cfo/ monthly-pl.md cash-forecast.md cmo/ campaign-brief.md ...（部門ごとにフォルダ分け） context/ company-overview.md ← 会社概要・事業説明 financials/ latest-pl.csv ← 最新PL budget-fy26.csv ← 予算 hr/ org-chart.md ← 組織図 marketing/ brand-guideline.md ← ブランドガイド
  • 2
    スキルファイルの書き方（CFO例）
    .claude/skills/cfo/monthly-pl.md
    ---
name: monthly-pl
description: 月次PL分析レポートを生成する
---

あなたはCFOとして月次PL分析を行います。

context/financials/latest-pl.csv を読み込み、
以下の観点で分析レポートを作成してください：

1. 売上・利益の前月比・予算比（差額と率）
2. 費用項目の増減トップ3の原因仮説
3. 注目すべき異常値・アラート項目
4. 翌月の着地見込みと施策提案

出力形式：
- エグゼクティブサマリー（3行）
- 部門別詳細テーブル
- アクション推奨リスト（優先度付き）
  • 3
    MCPサーバーを接続して実データを参照できるようにする
    # .claude/settings.json（主要MCPの接続例） { "mcpServers": { "notion": { "command": "npx", "args": ["-y", "@notionhq/notion-mcp-server"], "env": { "NOTION_API_KEY": "secret_xxx" } }, "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_TOKEN": "ghp_xxx" } }, "google-drive": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-gdrive"], "env": { "GDRIVE_CREDENTIALS": "./gdrive-creds.json" } } } }
  • 4
    CLAUDE.mdでグローバルコンテキストを設定する
    CLAUDE.md（プロジェクトルート）
    ## 会社コンテキスト
- 会社名：株式会社〇〇
- 事業：〇〇SaaS（B2B）
- 期末：3月末
- 使用ツール：Notion（ナレッジ）/ freee（会計）/ GitHub（開発）

## スキル使用の原則
- 財務データは context/financials/ の最新ファイルを参照する
- 社外提出資料は必ず「ドラフト」と明記する
- 個人情報を含む資料はローカル保存のみ。外部送信禁止
  • 5
    スキルを実行する
    # 月次PL分析を実行 /monthly-pl # OKR進捗レビュー（四半期ごと） /okr-review Q1 2026 # 求人票を生成 /jd-draft 職種：バックエンドエンジニア 必須：Python 3年以上、REST API設計経験 歓迎：FastAPI、クラウドインフラ（AWS/GCP）

注意点

  • スキルが多くなるほど管理コストが増える。最初は各部門3個程度に絞り、実際に使いながら増やすのが効率的。
  • 財務・人事データは機密性が高い。MCPで外部サービスに接続する際は、必要最小限のスコープのみ付与する。
  • スキルの品質はコンテキストファイルの鮮度に依存する。月次・四半期ごとに context/ フォルダのデータを更新する運用ルールを設ける。
  • AIが生成した経営判断・財務分析は最終的に人間が確認・承認する。特に対外的な資料（投資家向け等）は専門家レビューを経ること。
