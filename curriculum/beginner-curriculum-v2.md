# AI活用実践研修 初級編 カリキュラム v2

## 研修概要

| 項目 | 内容 |
|------|------|
| 研修名 | AI活用実践研修 初級編 |
| 対象者 | 一般ビジネスパーソン（職種不問・AI経験不問） |
| 形式 | オンラインライブ（同時双方向型通信訓練） |
| 時間 | 90分 × 全8回（総訓練時間12時間） |
| 使用ツール | Google Gemini、NotebookLM |
| 業種対応 | 3グループ（公共福祉系/技術専門系/顧客対応系）でお題を差し替え |
| 助成金 | 人材開発支援助成金（事業展開等リスキリング支援コース）対応 |

## コンセプト

> **「人を減らす」のではなく「今いる人のパフォーマンスを最大化する」**

AIを「もう一人の優秀なチームメイト」として活用し、一人ひとりの業務成果を高める。

---

## 卒業時の姿

> **自分の日常業務の中から「AIに任せられる部分」を見つけ出し、指示を出して下書きを作らせ、自分で仕上げるサイクルを回せる人になっている。**

### 5つの業務スキル

| # | スキル | 卒業時にできること | 対応する回 |
|---|--------|-----------------|-----------|
| 1 | **伝える力** | AIに的確な指示を出し、望む結果を引き出せる | 第1〜2回 |
| 2 | **書く力の拡張** | メール・議事録・報告書の下書きをAIに作らせ、自分で仕上げられる | 第3回 |
| 3 | **調べる力の拡張** | 膨大な資料や制度情報をAIに読ませて要点を把握できる | 第4〜5回 |
| 4 | **考える力の拡張** | AIと壁打ちしてアイデアを広げ、企画・提案の骨子を作れる | 第6回 |
| 5 | **まとめる力** | 調べた内容を整理し、人に伝わる資料にまとめられる | 第7〜8回 |

※ 第8回の総合演習では5つのスキルを複合的に使う（調べて→書いて→まとめる）

---

## 業種別グループ

| グループ | 対象業種 | お題の方向性 |
|---------|---------|-------------|
| **A. 公共・福祉系** | 社会福祉協議会、自治体、NPO、介護 | 議事録、行政報告書、広報文、住民案内 |
| **B. 技術・専門職系** | 建設・設計、製造業、士業 | 技術文書、法令・規格検索、品質報告書 |
| **C. 顧客対応系** | 小売・飲食・サービス、営業 | 顧客メール、SNS投稿、販促企画 |

カリキュラムの骨格（構成・学習目標・進行フロー）は全グループ共通。
差し替えるのは**演習のお題テキストのみ**。

---

## 全8回の構成

### 第1回：AIと話してみよう

| 項目 | 内容 |
|------|------|
| スキル | 伝える力 |
| ゴール | AIに質問し、回答を「使える／使えない」で判断できる。入力ルールを理解している |

**内容**:
- AIの仕組み（優秀だけど口が軽い新人）
- 入力ルール（安全/注意/禁止の3段階）
- Geminiを開いて実際に質問してみる
- 聞き方のコツ（具体的に・背景を伝える・やりとりを重ねる）
- AIの知ったかぶりを見破るクイズ

**業種別お題の差し替え**: 実践①のお題4つ、チャレンジ宿題

---

### 第2回：伝え方で結果が変わる

| 項目 | 内容 |
|------|------|
| スキル | 伝える力 |
| ゴール | 指示文の型（役割・背景・出力形式）を使って、狙い通りの回答を引き出せる |

**内容**:
- 指示文の3要素（役割指定・背景説明・出力形式の指定）
- 同じお題で指示文を段階的に改善するワーク
- 実際の業務シーンで指示文を書く演習
- 参加者同士で指示文と結果を共有

**業種別お題の差し替え**: 演習で使う業務シーンの例文

**発展トピック**: 例示を入れて精度を上げる技術、カスタム指示

---

### 第3回：業務文書を素早く作る

| 項目 | 内容 |
|------|------|
| スキル | 書く力の拡張 |
| ゴール | メール・議事録・報告書の下書きをAIで作り、自分の言葉で仕上げられる |

**内容**:
- ビジネスメールの下書き作成→推敲
- 会議メモから議事録のたたき台を作成
- 報告書の骨子作成
- トーン・長さ・敬語レベルの調整

**業種別お題の差し替え**: メールのシチュエーション、議事録の元メモ、報告書の種類

**発展トピック**: ボイスメモ→文字起こし→AI要約の一気通貫ワークフロー

---

### 第4回：情報を読み込ませて整理する

| 項目 | 内容 |
|------|------|
| スキル | 調べる力の拡張 |
| ゴール | 長い文書をAIに要約させ、比較表や論点整理ができる |

**内容**:
- PDFや長文をGeminiにアップロードして要約
- 複数の選択肢を比較表にまとめさせる
- 論点の構造化（メリット・デメリット、賛成・反対）
- 要約結果の正確性を確認する方法

**業種別お題の差し替え**: 読み込ませる資料の題材

**発展トピック**: 複数ファイルの横断分析、Google検索連携

---

### 第5回：深く調べてまとめる

| 項目 | 内容 |
|------|------|
| スキル | 調べる力の拡張 |
| ゴール | NotebookLM等で体系的に調査し、出典付きのレポートを作成できる |

**内容**:
- NotebookLMに複数資料を読み込ませてQ&A
- ディープリサーチでレポート形式の調査結果を取得
- 出典の確認方法（Perplexityの活用）
- 事実確認の実践

**業種別お題の差し替え**: 調査テーマ、読み込ませる資料

**発展トピック**: NotebookLMの音声要約機能

---

### 第6回：AIと一緒に考える

| 項目 | 内容 |
|------|------|
| スキル | 考える力の拡張 |
| ゴール | AIと壁打ちして企画の骨子を作り、弱点の指摘も受けられる |

**内容**:
- テーマを決めてAIとアイデア出し（10個→深掘り3個）
- 企画の骨子をAIと対話しながら構成
- 「この企画の弱点は？」「反論を想定して」を聞く
- 壁打ち結果を企画書の形にまとめる

**業種別お題の差し替え**: 企画のテーマ

**発展トピック**: 複数の立場を演じさせるブレスト

---

### 第7回：人に伝わる資料を作る

| 項目 | 内容 |
|------|------|
| スキル | まとめる力 |
| ゴール | 発表資料の構成をAIに作らせ、見せ方を仕上げられる |

**内容**:
- 発表資料のアウトライン作成
- 各スライドの要点を対話で詰める
- 画像生成機能の活用
- 「この資料を見た人にどう伝わるか」をAIにレビューしてもらう

**業種別お題の差し替え**: 発表のテーマ

**発展トピック**: コード実行でグラフ生成、Slides連携

---

### 第8回：自分の業務で実践する

| 項目 | 内容 |
|------|------|
| スキル | 5つの力すべて（複合） |
| ゴール | 自分の実際の業務課題を選び、調査→整理→作成を一気通貫で完遂する |

**内容**:
- テーマ選定＆計画（5分）
- 卒業課題：調査→情報整理→企画構成→文章化→資料骨子を60分で完成
- 成果発表（各参加者2〜3分）
- 全8回の振り返り＆今後のアクションプラン

**業種別お題の差し替え**: なし（参加者の実業務を使う）

## 各回共通の進行構造（90分）

| 時間 | パート | 内容 |
|------|--------|------|
| 0:00-0:05 | オープニング | 今日のゴール＋前回チャレンジの共有 |
| 0:05-0:20 | ミニ講義 | 今日のテーマの要点 |
| 0:20-0:35 | 実践① | 基本演習 |
| 0:35-0:40 | 休憩 | 5分 |
| 0:40-0:45 | ミニ講義② | 深掘りポイント |
| 0:45-1:10 | 実践② | 応用演習 |
| 1:10-1:20 | 共有＆振り返り | 成果共有・セルフチェック・チャレンジ宿題 |
| 1:20-1:30 | 発展トピック（任意） | 中級者向け＋次回予告 |

## 技術構成

```
materials/
├── presentation.html          ← 共通テンプレート（全回共通のベース）
├── session-01.html            ← 第1回（presentation.htmlベース）
├── session-02.html            ← 第2回
├── ...
├── topics/
│   ├── topics-A.json          ← A. 公共・福祉系のお題
│   ├── topics-B.json          ← B. 技術・専門系のお題
│   └── topics-C.json          ← C. 顧客対応系のお題
├── recap/
│   ├── session-01-recap.html  ← 第1回振り返り
│   └── ...
└── templates/
    ├── plan-template.md       ← 計画届テンプレート
    ├── attendance.md          ← 出席管理テンプレート
    └── supplementary.md       ← 補講運用ガイド
```

URLパラメータで業種グループを切り替え：
- `session-01.html?type=A` → 公共・福祉系のお題で表示
- `session-01.html?type=B` → 技術・専門系のお題で表示
- `session-01.html?type=C` → 顧客対応系のお題で表示
