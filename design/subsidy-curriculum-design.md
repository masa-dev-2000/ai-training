# 助成金対応カリキュラム設計 ― 業種オプション最小化戦略

## 前提

| 項目 | 内容 |
|------|------|
| 助成金 | 人材開発支援助成金（事業展開等リスキリング支援コース） |
| 申請主体 | 各受講企業（事業主） |
| あなたの立場 | 外部訓練機関（委託先）＋申請サポート |
| 対象業種 | 不特定（汎用設計が必要） |

---

## 業種ごとのカスタマイズが必要な箇所とそうでない箇所

### カスタマイズ不要（全業種共通にできる部分）

| 内容 | 理由 |
|------|------|
| AIの仕組み・基礎知識 | 業種によらない |
| 情報セキュリティ（OK/NGライン） | 業種によらない（個人情報・機密情報の扱い） |
| 指示文の型（役割・背景・出力形式） | 業種によらない汎用スキル |
| ハルシネーション体験・事実確認 | 業種によらない |
| AIツールの基本操作 | 業種によらない |
| 振り返り・セルフチェック | 業種によらない |

**→ カリキュラムの「骨格」（第1〜3回の大部分）は全業種共通で使える。**

### カスタマイズが必要な箇所

| 内容 | 理由 | カスタマイズの粒度 |
|------|------|-------------------|
| **演習のお題** | 「メール作成」は共通だが、具体的なシチュエーションは業種で変わる | 中 |
| **業務活用の具体例** | 「議事録」は共通だが、何の会議かは業種で変わる | 低 |
| **計画届の訓練目標** | 各企業の事業内容に合わせた記述が必要 | 高（企業ごと） |
| **第6〜8回の実践テーマ** | 実際の業務課題を使う回は業種で内容が変わる | 高 |

## オプションを最小化する設計

### 戦略：「3層構造」

```
┌──────────────────────────────────────┐
│  第3層：企業別カスタマイズ（計画届のみ） │ ← 企業ごとに作成
├──────────────────────────────────────┤
│  第2層：演習お題セット（業種別3パターン） │ ← 3種類で大半カバー
├──────────────────────────────────────┤
│  第1層：共通カリキュラム（全8回の骨格）  │ ← 全業種同一
└──────────────────────────────────────┘
```

### 第1層：共通カリキュラム（変更不要）

全8回の構成・学習目標・進行フローは全業種同一。
講義資料（presentation.html）もベースは同一。

### 第2層：演習お題セット（3パターン）

業種を大きく3つに分け、演習で使うお題を差し替える。

| パターン | 対象業種例 | 演習お題の方向性 |
|---------|-----------|----------------|
| A. 事務・管理系 | 社会福祉協議会、自治体、NPO、一般事務 | 議事録、広報資料、アンケート、住民向け案内文 |
| B. 営業・サービス系 | 不動産、小売、飲食、介護 | 顧客メール、提案書、クレーム対応、シフト管理 |
| C. 技術・専門職系 | 設計事務所、IT、建設、製造 | 技術文書、仕様書レビュー、規格調査、報告書 |

**具体的な差し替え箇所：**

| 回 | 共通部分 | 差し替え部分 |
|----|---------|-------------|
| 第1回 | AI基礎、OK/NG、操作 | 実践①のお題4つ、チャレンジ宿題 |
| 第2回 | 指示文の型 | 具体例のメール文、背景情報 |
| 第3回 | メール・議事録の作り方 | メールのシチュエーション、議事録の元メモ |
| 第4回 | 要約・比較・分析 | 読み込ませる資料の題材 |
| 第5回 | リサーチ・NotebookLM | 調査テーマ |
| 第6回 | 企画・ブレスト | 企画のお題 |
| 第7回 | プレゼン・画像 | プレゼンのテーマ |
| 第8回 | 総合演習 | 業務課題（参加者の実業務） |

**差し替え方法**: presentation.htmlの中でお題部分だけを変数化し、パターンA/B/Cで切り替え可能にする。
または、お題リストのJSONを外部化して読み込む。

### 第3層：企業別カスタマイズ（計画届）

各企業の計画届に書く内容は企業ごとに異なる。ただし**テンプレート化**できる。

**テンプレートの穴埋め項目：**

```
訓練名: AI活用実践研修 初級編
訓練目標: 【企業名】における業務のDX推進のため、
         従業員がAI（生成AI）を活用して【具体的な業務】を
         効率化するスキルを習得する。

対象者: 【企業名】の【部署名/全社員】（雇用保険被保険者）
訓練期間: 【開始日】〜【終了日】
総訓練時間: 【○○】時間
訓練実施方法: OFF-JT（同時双方向型通信訓練）
外部訓練機関: 【あなたの会社名】
```

**「訓練目標」の業種別テンプレート：**

| パターン | 訓練目標の書き方 |
|---------|----------------|
| A. 事務・管理系 | 「広報・事業企画・会議運営等の事務業務において、AIを活用した文書作成・情報整理・資料作成のスキルを習得し、業務の質と速度を向上させる」 |
| B. 営業・サービス系 | 「顧客対応・提案書作成・市場調査等の営業業務において、AIを活用した文書作成・情報収集・企画立案のスキルを習得し、顧客満足度と業務効率を向上させる」 |
| C. 技術・専門職系 | 「技術文書作成・規格調査・設計レビュー等の技術業務において、AIを活用した情報収集・分析・文書作成のスキルを習得し、業務品質と生産性を向上させる」 |

## 「趣味・教養」と見なされないための演習お題設計

### 第1回 チャレンジ宿題（現状→変更案）

**現状（業務と無関係）:**
- 献立を考えて
- お出かけスポット
- お礼のメッセージ

**変更案（業務関連＋汎用）:**
- 「明日の朝礼で1分間話す原稿を考えて。テーマは"最近の業務で工夫したこと"」
- 「上司に送る業務報告メールの下書きを作って。今週やったことを3つ箇条書きで」
- 「来月の会議のアジェンダ（議題一覧）を考えて。テーマは"業務改善"」

**ポイント**: 「業務で実際に使える成果物」が生まれるお題にする。

### 第1回 実践①のお題（現状→変更案）

**現状:**
- 朝礼スピーチ（ギリギリ業務寄り）
- 社会人1年目の知識（教養寄り）
- 歓迎会の企画（業務寄り）
- 旅行先のおすすめ（完全に趣味）

**変更案:**
- 「来週の朝礼で1分スピーチをします。テーマは"AIを使ってみた感想"で原稿を考えて」
- 「新しく入った後輩に、うちの部署の業務内容を3つのポイントで説明して」
- 「来月のチームミーティングのアジェンダを考えて。参加者5人、時間は1時間」
- 「上司への週報メールの下書きを作って。今週やったことを箇条書きで」

## 補講の仕組み

### 設計

| 方式 | 内容 |
|------|------|
| アーカイブ視聴 | 各回を録画し、欠席者にはアーカイブURLを共有 |
| 補講課題 | アーカイブ視聴後に簡単な課題を提出（セルフチェック＋演習1つ） |
| 出席認定 | アーカイブ視聴＋課題提出で「受講」として記録 |

### 助成金上の注意

- **同時双方向型通信訓練（ライブ）として計画**している場合、アーカイブ視聴が「受講」と認められるか要確認
- **安全策**: 補講は個別にライブで実施する（30分程度の短縮版）
- 補講の実施記録も同様に残す

### 受講率管理

| 全8回の場合 | 出席回数 | 受講率 | 判定 |
|------------|---------|--------|------|
| 8回中8回 | 12時間 | 100% | ✅ |
| 8回中7回 | 10.5時間 | 87.5% | ✅ |
| 8回中6回 | 9時間 | 75% | ❌ |

**→ 1回まで欠席OK。2回以上欠席したら補講必須。**

## 準備する成果物一覧

| # | 成果物 | 用途 | 業種カスタマイズ |
|---|--------|------|----------------|
| 1 | **共通カリキュラム** | 全企業共通の8回構成 | 不要 |
| 2 | **演習お題セット** A/B/C | 業種別の演習問題 | 3パターン |
| 3 | **計画届テンプレート** | 企業が労働局に提出 | 穴埋め式 |
| 4 | **訓練目標テンプレート** | 計画届の訓練目標 | 3パターン |
| 5 | **出席記録テンプレート** | 各回の出席管理 | 不要 |
| 6 | **実施状況報告書テンプレート** | 支給申請時に提出 | 不要 |
| 7 | **補講の仕組み説明資料** | 企業への案内用 | 不要 |
| 8 | **外部講師要件確認書** | 計画届に添付 | 不要 |
| 9 | **契約書・見積書テンプレート** | 企業との契約用 | 金額のみ変動 |

**→ 業種カスタマイズが必要なのは #2（3パターン）と #4（3パターン）のみ。他は全企業共通。**

---

## まとめ：オプション最小化の結論

**カスタマイズが必要なのは「演習お題」と「計画届の訓練目標」の2箇所のみ。**
**それぞれ3パターン（事務系・営業系・技術系）で大半の業種をカバーできる。**

講義資料（HTML）・カリキュラム骨格・出席管理・補講の仕組み・申請書類テンプレートは全て共通で使い回せる。
