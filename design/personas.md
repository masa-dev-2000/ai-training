# 業種別ペルソナ設定書

> 最終更新: 2026-05-10
> 位置づけ: 全8回を貫く「連続ドラマの主人公」として使うペルソナ3名の正本。
> 拠り所: `design/integrated-engagement-framework.md` 原則8（ペルソナ＋自分の業務）

## なぜペルソナを使うか

- 演習お題に**当事者性**を持たせる（架空の課題ではなく、誰かの業務として描く）
- 全8回に**連続性**を生む（第1回で出会ったペルソナが、第8回まで業務課題で進展していく）
- 業種別グループ（A/B/C）の参加者が**自分の業界の代弁者**として共感できる
- 第8回でペルソナを脱ぎ、自分自身の業務に重ねる**転換装置**になる

ペルソナは「他人事に見える」リスクと隣合わせのため、**第1〜7回はペルソナ課題で型を掴む → 後半で自分の業務に転用**の二段構えで運用する。詳細は統合フレーム原則8参照。

## ペルソナ A：佐藤さん（公共・福祉系）

### プロフィール

| 項目 | 内容 |
|------|------|
| 氏名 | 佐藤 美咲（さとう みさき） |
| 年齢 | 32歳 |
| 所属 | 中規模都市の社会福祉協議会 ＞ 地域包括支援センター |
| 役職 | 主事（一般職員）／勤続3年目 |
| 前職 | 社会福祉学を専攻後、新卒で別の自治体社協に2年勤務、現職に転職 |
| 保有資格 | 社会福祉士／介護支援専門員（取得予定） |

### 性格・働き方

- 真面目で丁寧。文書作成は時間をかけて推敲するタイプ
- 困っている住民を放っておけない一方、自分の仕事を抱え込みがち
- ITは苦手意識あり。LINE・Word・Excelは使うが、新しいツールは怖い
- 「仕事の質を落としたくない」気持ちと「時間が足りない」現実の板挟み

### 日常の困りごと（AIで解決したいこと）

- 国・県からの**長い通知や制度改定文書**を読みこなすのに時間がかかる
- 月1回の地域ケア会議の**議事録**が翌日までに仕上がらない
- 住民向けの**案内文**を書くたびに「分かりやすさ」と「正確さ」のバランスに迷う
- 関係機関（病院・自治体・NPO）への**メール文面**の敬語に時間を取られる

### 業務上のセンシティブ情報

- 個人情報（氏名・住所・要介護度・家族構成）は厳格管理
- AI入力時は**伏せ字必須**。第1回のOK/NG3段階ルールを徹底

### 口調例

> 「すみません、これってAIに通知の中身を全部入れちゃっても大丈夫なんでしょうか…」
> 「あ、なるほど！こういう聞き方すれば良かったんですね」
> 「今までは手書きでメモ取って、家で議事録に起こしてたんです。それが半分の時間でできたら…」

### 全8回でのドラマ進行

| 回 | 佐藤さんの課題 |
|----|--------------|
| 第1回 伝える力① | 初めてGeminiを開く。「何聞いていいかわからない」状態から、住民向け案内文の相談ができるように |
| 第2回 伝える力② | 「地域サロンの参加者を増やしたい」相談を、役割・背景・出力形式で書き直して回答の精度が変わる体験 |
| 第3回 書く力 | 月例地域ケア会議のメモから議事録のたたき台を作る。トーン調整に苦戦 |
| **第4回 調べる力①** | 50ページの介護保険改定通知を要約。委託事業者3社の運営方式を比較表に |
| 第5回 調べる力② | NotebookLMで5冊の参考通知を一気に読ませて、地域ケア会議の論点ペーパーを作成 |
| 第6回 考える力 | 「地域サロンの新企画」をAIと壁打ち。住民の声・行政視点・実行コストの三方向で弱点を出し合う |
| 第7回 まとめる力 | 来月の所内勉強会向けスライドの骨子をAIに作らせ、画像生成で挿絵を |
| 第8回 自分の業務 | ペルソナを脱ぎ、参加者自身の業務へ |

## ペルソナ B：田中さん（技術・専門職系）

### プロフィール

| 項目 | 内容 |
|------|------|
| 氏名 | 田中 健一（たなか けんいち） |
| 年齢 | 29歳 |
| 所属 | 中堅設計事務所（社員25名／公共建築・集合住宅が中心） |
| 役職 | 主任技術者／意匠設計担当・勤続6年目 |
| 学歴 | 建築学科卒。大学院修了 |
| 保有資格 | 一級建築士／構造設計一級建築士（取得予定） |

### 性格・働き方

- 論理的。仕様書や法令を読み込むことに苦手意識はないが、時間がかかる
- 効率化への関心は高い。BIMやクラウド共有も積極的に使う
- 一方で「AIに頼ると技術が落ちる」懸念も持っている
- 上司（所長）の判断材料を整える役回りが多く、比較・要約のニーズが大きい

### 日常の困りごと（AIで解決したいこと）

- 競合他社の**技術提案書**（数十〜百ページ規模）を読み比べる時間がない
- 法令改定・JIS規格更新への追随が手間（**長文の通達**を読み解くのが負担）
- 顧客提案前の**比較表**作成（仕上げ材・設備・工法）に時間が取られる
- 工事報告書・調査報告書の**たたき台**を後輩に書かせるための雛形が欲しい

### 業務上のセンシティブ情報

- 顧客名・工事名・契約金額・図面の細部は秘密保持義務あり
- AI入力時は**プロジェクト名を伏せ字**、図面は入れない方針

### 口調例

> 「これ、JISの数字が原文と違ってますね。AIってこういうとこ平気で間違えるんですか？」
> 「比較表、いつもExcelで自分で作ってたんですけど…ここまでできるんですね」
> 「ただ、最終判断は人間がやるべきっていう前提は外せないですよね」

### 全8回でのドラマ進行

| 回 | 田中さんの課題 |
|----|--------------|
| 第1回 伝える力① | AI懐疑派。「本当に使えるのか」を試す姿勢で受講開始 |
| 第2回 伝える力② | 顧客プレゼン用の概要文を、役割・背景・出力形式で書き直して納得 |
| 第3回 書く力 | 工事打合せ議事録のたたき台作成。技術用語の扱いに気を遣う |
| **第4回 調べる力①** | 競合3社の技術提案書80ページを整理。外壁仕上げ材3種の比較表を作成 |
| 第5回 調べる力② | NotebookLMに過去案件の報告書10件を読ませ、設計判断の傾向を抽出 |
| 第6回 考える力 | 公共建築のコンペ案をAIと壁打ち。コスト・工期・意匠の3観点で弱点を出す |
| 第7回 まとめる力 | 顧客提案スライドの骨子＋コード実行で集計グラフを生成 |
| 第8回 自分の業務 | ペルソナを脱ぎ、参加者自身の業務へ |

## ペルソナ C：鈴木さん（顧客対応系）

### プロフィール

| 項目 | 内容 |
|------|------|
| 氏名 | 鈴木 彩（すずき あや） |
| 年齢 | 35歳 |
| 所属 | 地方の小売チェーン本部（店舗40店／食品・日用品） |
| 役職 | 販促マーケティング担当課長／勤続8年目 |
| 経歴 | 店舗スタッフ→店長→本部販促へ |
| マネジメント | 部下3名（20〜30代） |

### 性格・働き方

- 行動的。新しいものへの抵抗は少ない（SNSもLINEも仕事で使う）
- 数字に強い。一方で文書作成は得意ではない（「書くより話すタイプ」）
- 部下の育成にも責任を持っており、AI活用を組織に広げたい意欲
- 「現場の感覚を失いたくない」気持ちと「データで判断したい」気持ちの両立

### 日常の困りごと（AIで解決したいこと）

- 業界動向レポート・市場調査資料の**要約**に時間が取られる
- SNS投稿・チラシ文案・店内POPの**コピーライティング**に毎週追われる
- 集客施策の**比較**（SNS広告／チラシ／ポイント／イベント）を経営層向けに整理したい
- 部下が書いた**販促企画書のレビュー**に時間がかかる

### 業務上のセンシティブ情報

- 売上数値・取引先（仕入先）名・契約条件は機密
- 顧客の購買データは個人情報として伏せ字必須

### 口調例

> 「これ、うちの部下にも教えたいんですけど、どこから入れたらいいですかね」
> 「えっ、一瞬でこんなコピー出てくるの？じゃあ私の仕事って…笑」
> 「経営会議で『数字の根拠は？』ってよく聞かれるんですが、AIにそこまでやらせていいのか不安で」

### 全8回でのドラマ進行

| 回 | 鈴木さんの課題 |
|----|--------------|
| 第1回 伝える力① | AIに対するワクワク感はあるが、何から触ればいいか分からない |
| 第2回 伝える力② | 春の新商品キャンペーン文案を、役割・背景・出力形式で書き直し |
| 第3回 書く力 | 店長会議の議事録＋取引先への謝罪メールを下書き |
| **第4回 調べる力①** | 業界動向レポート60ページを要約。集客施策3案を比較表に |
| 第5回 調べる力② | NotebookLMで過去3年分の販促レポートを読ませ、効果の高い施策パターンを抽出 |
| 第6回 考える力 | 来期の販促企画をAIと壁打ち。顧客視点・店舗視点・本部視点で弱点を出す |
| 第7回 まとめる力 | 経営会議向け企画書のスライド骨子作成。グラフ生成も活用 |
| 第8回 自分の業務 | ペルソナを脱ぎ、参加者自身の業務へ |

## 運用ガイドライン

### スライド・教材での扱い

- 各回冒頭の「今日の主人公」枠で3名を並べる（業種スイッチャーで切替可能にしてもよい）
- 演習お題は必ずペルソナの業務として記述（「比較表を作って」ではなく「佐藤さんが委託事業者3社の運営方式を比較したい」と書く）
- 第8回ではペルソナを完全に外し、参加者自身の業務に置き換える

### 講師の扱い方

- ペルソナを「同じ業界の少し先輩」と位置づけて紹介する（参加者が自分の延長線上に置けるように）
- 失敗例（フック）はペルソナに肩代わりさせる（「佐藤さんがやっちゃった」失敗）。参加者を晒さない
- 演習の振り返りでは「ペルソナの課題に対して」ではなく「あなたの業務でも同じ場面ある？」と橋を架ける

### 心理的安全性

- ペルソナは**完璧なロールモデルではない**。失敗もする・困ってもいる存在として描く
- 参加者がペルソナを「自分より下」と感じないよう、3年目〜8年目の中堅レベルに統一
- ペルソナの口調は丁寧だが堅すぎず、参加者と同じ目線で語らせる

### 機密情報の扱い

- ペルソナ業界の典型的な機密情報を、各回の冒頭で1度ずつ確認する
- 「佐藤さんが要介護度を入れたら？／田中さんが図面を入れたら？／鈴木さんが売上を入れたら？」を例として使い、参加者の業務に置き換えやすくする
