| **状況を伝える** | 誰に・何のために・どんな立場で | 「役割」と「背景」を統合 |

| **形を指定する** | 箇条書き、表、文字数、トーン | 変更なし |

## 5つの応用テクニック

### テクニック1：手順を分けて指示する（ステップ分割）

1回の指示で複雑なことを頼まず、手順を分けて書く。

**悪い例**: 「会議の内容をまとめて、問題点を分析して、改善案を出して」

**良い例**:
```
以下の手順で進めてください。
1. まず会議メモの内容を箇条書きで整理してください
2. 次に、そこから見える問題点を3つ挙げてください
3. 最後に、各問題点の改善案を1つずつ提案してください
```

### テクニック2：見本を渡す（テンプレート指示）

「このような形で書いてください」と見本を1つ添える。

```
以下の形式で、今月の業務報告を書いてください。

【見本】
■ 主な成果
・○○を実施し、△△の結果を得た
■ 課題
・○○について改善が必要
■ 来月の予定
・○○を予定
```

### テクニック3：AIに質問させる（インタビュー形式）

自分が情報を整理できていない時に有効。

```
業務改善の提案書を作りたいのですが、
必要な情報を私に1つずつ質問してください。
全ての情報が揃ったら、提案書を作成してください。
```

### テクニック4：条件と制約を明示する

AIが勝手に判断しないように、やってほしくないことも伝える。

```
この文章を校正してください。
- 内容は変えないでください
- 誤字脱字だけを直してください
- 修正した箇所は【 】で囲んで教えてください
```

### テクニック5：出力を評価させる（セルフチェック指示）

AIに自分の出力を振り返らせる。

```
上記の提案書を、以下の観点で自己評価してください。
- 論理の飛躍はないか
- 実現可能性はあるか
- 読む人が理解できる表現になっているか
問題があれば修正版も出してください。
```

## やりとり戦略（3段階）

| 段階 | やること | プロンプト例 |
|------|---------|-------------|
| 1. 大枠を出す | まず全体の骨子を作らせる | 「○○の提案書の骨子を箇条書きで出して」 |
| 2. 修正する | 気になるところを直す | 「2番目の項目をもっと具体的にして」「全体をもう少し短くして」 |
| 3. 仕上げる | 最終形にまとめる | 「以上を踏まえて、完成版を書いて」 |

**失敗した時の立て直し方**:
- 的外れな回答 → 「そうではなくて、○○という意味です」と明確に否定＋再説明
- 長すぎる → 「200文字以内で書き直してください」（具体的な数字で）
- 敬語がおかしい → 「取引先の部長に送れるレベルの丁寧な敬語で書き直してください」
- 何度やってもダメ → 「新しい会話を始めて、最初から指示し直す」（リセットが有効な場合も）

## 実践②：実務レベルの総合演習（20分）
