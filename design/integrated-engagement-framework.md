# 統合エンゲージメント・フレームワーク（書籍×映画×教育）

> 最終更新: 2026-05-10
> 対象: AI活用研修 初級編・90分×8回・10〜20名・オンラインライブ・社会人
> 位置づけ: 既存の `group-lecture-engagement.md`（書籍・映画由来の7原則）に、教育現場のノウハウ（`調査レポート参照`）をマージし、3領域を**1つの設計言語**に統合する。本ファイルが今後の各回設計の正本となる。

---

## 0. なぜ統合するか

3領域は強みが異なる。
- **映画**: 90分というスパンを「飽きさせず1本の作品にする」骨格（三幕構成・クリフハンガー）
- **書籍**: 「読み手に章を進ませる」誘引（冒頭フック・章立て・連続ドラマ）
- **教育**: 「理解を定着させる／学習者全員を巻き込む」装置（想起練習・出口チケット・I do/We do/You do・ピア・インストラクション・アンドラゴジー）

3つは**目的レイヤーが違う**ため、単純に並べると冗長になり、矛盾する場面では片方を捨てがち。統合フレームでは**役割分担を明示**して、衝突は解消オプション付きで残す。

---

## 1. 統合フレームの全体構造（5層モデル）

3領域を5つの層に再整理する。各層はそれぞれの領域から「最も効くもの」を選び、必要に応じて合成する。

| 層 | 役割 | 主担当領域 | 採用要素 |
|----|------|----------|---------|
| **L1 骨格層**（90分の構造） | 全体のリズムを決める | 映画 | 三幕構成 |
| **L2 進行層**（各シーンの中身） | 学習を成立させる | 教育 | ガニェ9教授事象 / I do・We do・You do |
| **L3 注意層**（引き込み） | 集中を切らさない | 書籍×映画 | 冒頭フック / 章タイトル / 二重符号化 / ピア投票 |
| **L4 定着層**（記憶に残す） | 研修後に効く | 教育 | 想起練習 / 出口チケット / インターリービング |
| **L5 横串層**（全8回を貫く） | 連続性と当事者性 | 書籍×教育 | 連続ドラマ（ペルソナ）／アンドラゴジー（自分の業務） |

※ クリフハンガーは L1（骨格）と L4（定着）の橋渡し役として独立に扱う。

## 2. 統合10原則

既存7原則を再編し、教育学由来の3原則を追加して**10原則**に整理。

### L1：骨格層

**原則1：90分は三幕構成で書く（マクロ骨格）**
- 第1幕（つかみ・0:00-0:25）／ 第2幕（葛藤と山場・0:25-1:10）／ 第3幕（解決と引き・1:10-1:30）
- 山場は第2幕の後半に置く。終盤に向けて高揚する。

**原則2：各シーンはガニェ9事象 + I do/We do/You do で埋める（ミクロ進行）**
- ミニ講義1本＝「①注意 → ②目標 → ③前提想起 → ④提示（I do）→ ⑤指針 → ⑥練習（We do→You do）→ ⑦フィードバック」
- 三幕がマクロの骨、9事象＋足場かけがミクロの肉。

### L2：注意層

**原則3：冒頭60秒は「衝撃の事例」で掴む**
- 結論や目次から始めない。失敗AI回答のスクショ・before/after画像を見せる。
- 二重符号化（言語＋視覚）原則を遵守し、テキストとビジュアルを必ずペアで。

**原則4：群衆ダイナミクスはピア・インストラクション化する**
- 単発の予測投票ではなく、**①個人投票→②ペア議論→③再投票→④AIに投げて答え合わせ** の4段。
- Think-Pair-Share（個人30秒→ペア2分→1組共有）を発問の標準テンプレに。
- コールド・コールではなく **Warm Call**（事前にチャットで指名予告）を使う。

**原則5：シーン切替で緩急をつける**
- 90分を8〜10シーンに分割。隣接シーンは必ずモードを変える。
- 講師話／ライブデモ／個人ワーク／ペアワーク／チーム対抗／全体投票／全体共有／チャット一斉投稿、を循環させる。

### L3：定着層

**原則6：想起と転移のループを全8回に通す**（**新規・教育学由来**）
- 各回冒頭3分：前回想起クイズ（Mentimeter / Zoom投票）
- 各回終盤2分：出口チケット（Googleフォーム or チャット）
- 翌日 / 3日後 / 1週間後にミニ課題を配信
- カリキュラム後半（第5・8回）でインターリービング演習を入れ、過去スキルを混合して問う。

**原則7：完成例（worked example）→変形 で教える**（**新規・教育学由来**）
- 初級編は「白紙から考えさせる」ではなく「完成された指示文を提示し、変形させる」が鉄則。
- 認知負荷理論：スライド本文を講師が読み上げない（冗長性原則）。文字＋ナレーションよりも図＋ナレーション。

### L4：横串層

**原則8：連続ドラマで通し、最後に自分に重ねる**
- 第1〜7回は業種別ペルソナ（佐藤さん／田中さん／鈴木さん）の業務課題として演習。
- 第8回でペルソナを脱ぎ、自分自身の業務に切り替える（アンドラゴジー：自己関連性）。
- 各回冒頭で**「なぜ今日これをやるか」**を必ず明言（ガニェ第2事象＋アンドラゴジー）。

### L5：橋渡し

**原則9：休憩前と回末はクリフハンガーで終える**
- 休憩前：「実はこの指示文に1つ罠がある。休憩後に答え合わせ」
- 回末：「次回はこれを1行加えるだけで結果が激変する。何だと思う？」
- ただし**回末は出口チケット → クリフハンガー**の順で並べる（原則6と両立、後述コンフリクト1）。

**原則10：メタ情報は1箇所に集約し、常時表示する**
- 「現在の章タイトル」「90分中の現在地」「全8回中の現在地」「今日の構造図」を画面トップに固定。
- スライド本体は1スライド1主題（認知負荷理論：外在的負荷を削減）。

## 3. コンフリクトポイントと解消オプション

3領域が**真正面から衝突する10箇所**を抽出し、それぞれ解消オプションを2〜3案ずつ示す。各回設計時にどのオプションを採るかを明示すること。

### コンフリクト1：「閉じるのか開いたまま終えるのか」
- **映画派**: クリフハンガーで未解決のまま終える（次回への引き）
- **教育派**: 出口チケットで今日の学びを言語化させて閉じる
- **解消オプション**:
  - **A（推奨）**: **出口チケット → クリフハンガー** の二段構え。「今日学んだこと1行＋明日試すこと1行」を書かせた直後、講師が「次回はこれを覆す技を教える」と引く。閉じてから開く。
  - B: クリフハンガーを出口チケットの設問に組み込む（「次回までに自分なりの仮説を1行書いて」）。一体化させて時間節約。
  - C: 偶数回は出口チケット重視、奇数回はクリフハンガー重視で交互に。

### コンフリクト2：「冒頭はフックか目標提示か」
- **映画/書籍派**: 60秒の衝撃シーンで掴む。結論や目次から始めない。
- **教育派**: ガニェ第2事象「学習目標の提示」を最初に行う。アンドラゴジー「なぜ学ぶか」も冒頭。
- **解消オプション**:
  - **A（推奨）**: **フックの中に目標を埋め込む**。失敗例を見せた直後に「だから今日はこれを身につける」と接続。フック＝目標の生々しい予告として設計する。
  - B: 0:00-0:01 フック → 0:01-0:03 想起クイズ（前回） → 0:03-0:05 ゴール提示。順序で並立。
  - C: グループによって変える。AI初学者にはフック先行、上級者には目標先行。

### コンフリクト3：「競争（チーム対抗）か協同（ジグソー）か」
- **映画派**: チーム対抗・指示文選手権で熱狂を作る
- **教育派**: 心理的安全性が下がる競争より、知識構成型ジグソー法のような協同が学習に効く
- **解消オプション**:
  - **A（推奨）**: 競争の対象を「人 vs 人」にしない。**チーム vs AI出力の質**で競わせ、判定もAI出力の質で行う。勝敗が個人攻撃にならない。
  - B: 第2回・第4回・第6回は競争モード、第3回・第5回・第7回は協同モード（ジグソー）で交互に配置。
  - C: 競争はスパイス、協同を基本モードとする（チーム対抗は1講義1回まで）。

### コンフリクト4：「ペルソナ（架空）か自分の業務か」
- **書籍派**: 連続ドラマ化で当事者を架空のペルソナに投影
- **教育派（アンドラゴジー）**: 大人は自分の経験を活かしたい。架空課題は他人事になりやすい。
- **解消オプション**:
  - **A（推奨）**: 第1〜7回は**ペルソナ＋自分の業務の二段演習**。前半でペルソナ課題を全員で解く（共通基盤）→ 後半で自分の業務に転用する（個人最適）。第8回はペルソナを完全に外す。
  - B: ペルソナをやめて、第1回冒頭で各自に「今後7回の演習で使う自分の業務テーマ」を1つ選んでもらう（アンドラゴジー徹底）。連続ドラマは捨てる。
  - C: ペルソナを「同じ業界の先輩」と位置づけ、参加者が「自分のミニ版」として自然に重ねられるように設計する。

### コンフリクト5：「物語のリズム（緩急）か学習のリズム（一定の足場かけ）か」
- **映画派**: 90分の中で緊張と緩和を意図的に揺らす
- **教育派**: I do→We do→You do は**毎回同じ順序**で繰り返すから定着する
- **解消オプション**:
  - **A（推奨）**: 各シーン内部は I do→We do→You do を厳守（学習リズム維持）、シーン間の切替で緩急をつける（物語リズム維持）。**ミクロは規則的、マクロは不規則**。
  - B: 第1幕は規則的（学習リズム優先）、第2幕で揺さぶる（物語リズム優先）。役割分担。
  - C: 学習リズムを優先し、物語リズムは諦める（おとなしいが安定）。

### コンフリクト6：「メタ情報の常時表示（書籍式）か認知負荷の削減（教育式）か」
- **書籍/林修式**: 章タイトル・進捗バー・構造図を常時表示で安心感を提供
- **教育（認知負荷理論）**: 画面情報を増やすと外在的負荷が上がり、学習が阻害される
- **解消オプション**:
  - **A（推奨）**: メタ情報は**画面トップの細い帯1つ**に集約（高さ44px固定、既存実装を踏襲）。スライド本体は1主題1スライド。両立可能。
  - B: メタ情報は**Tabキーやキーで折りたたみ可能**にする。普段は隠し、講師が要所で出す。
  - C: 構造図は紙の補助資料として配布し、画面には載せない。

### コンフリクト7：「想起クイズ（教育）か冒頭フック（映画）か。どちらを冒頭2分に置くか」
- **教育派**: 前回想起→定着強化→今日へ接続が王道
- **映画派**: 衝撃のフックでまず注意を奪う
- **解消オプション**:
  - **A（推奨）**: **フックを「前回学んだ技を試した結果」のbefore/after画像にする**。フックと想起を一体化。「前回これを学んだ。Aさんが試したらこうなった。すごい？それとも罠がある？」。
  - B: 0:00-0:01 フック → 0:01-0:03 想起クイズ → 0:03-0:05 ゴール の順で時間配分（コンフリクト2と同案）。
  - C: 第1回はフック、第2回以降は想起クイズで開始（連続性を優先）。

### コンフリクト8：「インターリービング（混ぜる）かカリキュラム整然性（1回1スキル）か」
- **教育派**: 同種の問題を混ぜたほうが転移する
- **カリキュラムv2**: 1回1スキルで整然と進む（伝える→書く→調べる→考える→まとめる）
- **解消オプション**:
  - **A（推奨）**: 各回は1スキル中心を維持しつつ、**チャレンジ宿題と第8回でインターリービング**を効かせる。前回想起クイズも複数回分を混ぜる。
  - B: 第1〜7回はブロック練習、第8回をインターリービング演習として設計（最終回が応用編になる）。
  - C: 各回終盤の「実践④」を過去スキル混合枠にする。

### コンフリクト9：「コールド・コール（教育）か沈黙耐性のないオンライン（現実）か」
- **教育派**: 全員が思考準備状態を保つにはコールド・コール
- **オンライン現実**: 急な指名はカメラオフの社会人を萎縮させる
- **解消オプション**:
  - **A（推奨）**: **Warm Call**（事前予告）＋ Think-Pair-Share後の指名。「ペアで話した内容を○○さん、1つだけ教えて」と限定。
  - B: 指名はせず、チャット投稿の中から講師が読み上げる方式に統一（顔出し不要）。
  - C: 1回目は希望者のみ、2回目以降は事前に決めた指名ローテで回す。

### コンフリクト10：「90分の物語密度（映画）か学習の余白（教育）か」
- **映画派**: 8〜10シーンに分割し、ノンストップで進む
- **教育派**: ワーキングメモリ容量に限界。詰め込みは逆効果。worked exampleと省略で負荷を下げる。
- **解消オプション**:
  - **A（推奨）**: シーン数は維持（飽き対策）、**1シーン内の情報量を半減**（worked example化・1主題1スライド）。密度ではなく**転換頻度**で飽きを防ぐ。
  - B: 90分を75分に縮小し、15分を「沈黙の練習時間」として確保（負荷削減優先）。
  - C: シーンを6つに減らし、1シーン15分に。物語密度を捨てる。

## 4. 実装パッケージ（軽量／標準／重厚）

オプション選択を毎回ゼロから判断すると運営が回らないため、3つのプリセットを用意。

### 軽量パッケージ（最低限）
- 採用原則: 1（三幕）／ 3（フック）／ 5（モード切替）／ 6（想起・出口チケットのみ）／ 9（クリフハンガー）
- 不採用: ジグソー、I do/We do/You do、Warm Call、構造図常時表示
- 適用場面: **講師1名運営・テスト講義・第1回の改訂初版**
- コンフリクト解消: 全てA案で固定

### 標準パッケージ（推奨・10原則すべて）
- 採用原則: 1〜10すべて
- 適用場面: **本番運営（10〜20名・複数社共通）**
- コンフリクト解消: 1A / 2A / 3A / 4A / 5A / 6A / 7A / 8A / 9A / 10A
- 第8回はインターリービング・自分業務化の特別運用

### 重厚パッケージ（フラッグシップ）
- 標準＋ジグソー法（第4回限定）＋授業研究（毎回後の30分検討会）＋スパイス回（第6回でオプション3B：競争／協同を交互運用）
- 適用場面: **企業の旗艦研修・講師2名以上運営**

## 7. 各回設計時のチェックリスト（統合版）

| 観点 | チェック項目 |
|------|------------|
| L1 骨格 | 三幕構成として書かれているか／山場が第2幕後半にあるか |
| L2 進行 | 各シーンが I do→We do→You do の順か／ガニェ9事象が満たされているか |
| L3 注意 | 冒頭60秒のフックがあるか／二重符号化（図＋言語）か／予測投票がピア・インストラクション化されているか |
| L4 定着 | 想起クイズと出口チケットが入っているか／worked exampleで提示しているか／3日後ミニ課題が予約されているか |
| L5 横串 | ペルソナが今回の主人公として登場しているか／自分の業務に転用するパートがあるか／「なぜ今日これか」が明言されているか |
| 橋渡し | 休憩前と回末がクリフハンガーで終わっているか／出口チケット→クリフハンガーの順か |
| メタ | 章タイトル・進捗バー・全8回中の現在地が常時表示されているか／1スライド1主題か |
| 運営 | コールド・コールでなくWarm Callか／競争は人vs人でなくvsAI出力の質か |
| 認知負荷 | スライド本文を講師が読み上げない構成か／情報密度が過剰でないか |
