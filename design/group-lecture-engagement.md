# 団体講義向け 進行設計原則

> 最終更新: 2026-05-10
> 対象: 全8回・各90分・オンラインライブ・10〜20名想定
> 位置づけ: カリキュラムv2の骨格は維持したまま、**各回の進行（タイムライン・演出）を団体講義向けに作り直す**ための設計原則集。

## 課題認識

現状の各回設計（第1回〜第4回の設計書、`materials/session-01/presentation.html`）は、テスター講義（1名相手）を前提に最適化されている。
- 講師が個別に話を引き出す進行
- 個人ワーク中心、共有はあっても全員1巡型
- 90分の中での緩急が「講義→演習→共有」の素直な繰り返しのみ
- 物語的な引き・章立てが弱い

10〜20名の団体に同じ進行を当てると、間延び・私語・離脱が出やすい。
カリキュラムの**学習目標は変えず**、進行の演出だけを団体向けに作り直す。

## 設計原則（7つ）

### 原則1：90分を「三幕構成」として書く

映画・小説の三幕構成を90分に当てる。各幕の終わりに**転換点**を置く。

| 幕 | 時間 | 役割 | 終わり方（転換点） |
|----|------|------|------------------|
| 第1幕：つかみ | 0:00-0:25 | 問いの提示・基礎理解 | 「ではこの状況、あなたならどうAIに頼む？」と問いを残して休憩 |
| 第2幕：葛藤 | 0:30-1:10 | 演習・つまずき・発見 | 一番盛り上がる山場（チーム対抗・予測投票など）を後半に置く |
| 第3幕：解決 | 1:10-1:30 | 統合・持ち帰り・次回への引き | 次回への**クリフハンガー**で締める |

**禁則**: 各幕内で15分以上同じモード（講師が話し続ける／個人ワーク／全体共有）を続けない。

### 原則2：冒頭60秒のフックを必ず置く

ミニ講義の前に、**「えっ」と思わせる素材**を1つ提示してから今日の問いを立てる。

- 失敗例の生々しいスクショ（情報漏洩、嘘回答、的外れ要約）
- 同じ質問でAIが正反対の回答をした2画面比較
- 前回の参加者が引き出した秀逸な回答 vs ありがちな回答

書籍でいう「冒頭1ページで読者を掴む」手法。**結論や目次から始めない**。

### 原則3：群衆ダイナミクスを各回に最低2つ仕込む

1人ではできない、団体だからこそ機能する仕掛けを必ず入れる。

| 仕掛け | 概要 | 想定時間 |
|--------|------|---------|
| 予測投票 | 「AIは次にどう答える？ A/B/Cで挙手（or Zoom投票）」→ 答え合わせ | 2〜3分 |
| 指示文選手権 | チーム対抗で同じお題に指示文を作り、結果のクオリティを全員投票 | 10〜15分 |
| 指示文交換 | ペアで指示文を交換して相手のAIに投げる→他人の指示で動くAIを観察 | 8〜10分 |
| ライブ添削 | 参加者の指示文を講師がその場でAIに投げ、改善過程を全員で観る | 5〜10分 |
| 同時タイピング | 全員一斉に同じお題を投げ、結果のばらつきを共有 | 5分 |
| ベスト共有 | 「うまくいった人だけ手を挙げて」→ 1〜2名に画面共有してもらう | 3〜5分 |

オンラインの制約（カメラオフ・反応薄）を逆手に取り、**チャット・投票・リアクションボタン**を多用する。

### 原則4：シーン切替で緩急をつける（モードのリズム）

90分を10〜15分の「シーン」に分け、隣り合うシーンは**必ずモードを変える**。

```
講義 → 個人ワーク → 講義 → 個人ワーク …  ← NG（リズムが平板）
講義 → 予測投票 → ペアワーク → 全体共有 → 個人ワーク → チーム対抗 → 講師ライブデモ → 振り返り  ← OK
```

モードの種類:
- 講師がしゃべる（10分以内）
- 講師がライブで操作を見せる（5分以内）
- 個人ワーク（10分以内）
- ペア／3人組ワーク
- チーム対抗
- 全体投票・挙手
- チャット一斉投稿
- 全体共有（代表者）

### 原則5：横串の主人公（連続ドラマ化）

全8回を通して、**業種別グループごとの「主人公」（架空のペルソナ）** を1名立て、毎回その人の業務課題が進展する連続ドラマにする。

- グループA：地域包括支援センターに転職した3年目職員「佐藤さん」
- グループB：中堅設計事務所の若手主任「田中さん」
- グループC：地方の小売チェーン本部の販促担当「鈴木さん」

各回のお題はこのペルソナの業務課題として登場。第8回（自分の業務で実践）でペルソナを脱ぎ、自分自身に重ねる構造。
これにより**毎回の話に連続性**が生まれ、欠席による離脱を減らす。

### 原則6：休憩前と回末はクリフハンガーで終える

連続ドラマの「次回予告」と同じ。休憩・終了の直前に**未解決の問い**を残す。

- 休憩前：「ここまでの指示文、実は1つ落とし穴があります。休憩のあと答え合わせします」
- 回末：「次回はこの指示文を、たった1行加えるだけで結果が激変させます。何を足すと思いますか？」

**禁則**: 「お疲れさまでした、また来週」だけで終わらせない。

### 原則7：可視化された「進捗バー」と「章タイトル」

書籍の章立て・映画のチャプター表示を真似る。スライドに常時表示するもの:

- 全8回中の現在地（例：●●●○○○○○ 第3回）
- 90分のうち現在地（チャプター進捗バー）
- 今のシーンのタイトル（例：「第2幕・指示文選手権」）

参加者が**残り時間と展開を見通せる**と、長丁場でも集中が切れにくい（書籍の「あと何ページ」効果）。

## 90分タイムライン雛形（団体・10〜20名）

| 時間 | チャプター | モード | 内容 |
|------|----------|--------|------|
| 0:00-0:02 | プロローグ | フック | 衝撃の失敗例・前回の名作回答 |
| 0:02-0:05 | オープニング | 講義 | 今日の問いを宣言（ゴール提示） |
| 0:05-0:15 | 第1幕：基礎 | 講義+ライブデモ | ミニ講義（5分）→ 講師がAI操作を見せる（5分） |
| 0:15-0:25 | 第1幕：練習 | 個人ワーク+全体投票 | 各自お試し→「うまくいった人挙手」→ 1名画面共有 |
| 0:25-0:30 | 休憩前フック | 講義 | クリフハンガー：「実はこの指示文に落とし穴が…」 |
| 0:30-0:35 | 休憩 | — | 5分 |
| 0:35-0:45 | 第2幕：応用 | 講義+予測投票 | 落とし穴の答え合わせ→ 応用テクニックの予測投票 |
| 0:45-1:05 | 第2幕：山場 | チーム対抗 | 指示文選手権（4〜5チーム×4分作成→投票） |
| 1:05-1:15 | 第2幕：余韻 | ペアワーク | 勝ったチームの型を真似て自分のお題に適用 |
| 1:15-1:25 | 第3幕：統合 | 全体共有+講義 | 代表3名共有→今日の3点まとめ→セルフチェック |
| 1:25-1:30 | エピローグ | 次回予告 | クリフハンガーで終える |

※ 90分内で**8〜10シーン**に区切る。各シーン10分以下。

## 各回設計時のチェックリスト

各回の進行を書き終えたら、以下に全て○がつくか確認する。

- [ ] 冒頭60秒に「えっ」と思わせるフックがある
- [ ] 三幕構成として、各幕の終わりに転換点がある
- [ ] 群衆ダイナミクス（団体ならではの仕掛け）が2つ以上ある
- [ ] 同じモードが15分以上連続していない
- [ ] 休憩前と回末がクリフハンガーで終わっている
- [ ] 業種別ペルソナが今回の主人公として登場している
- [ ] スライドに進捗バー・章タイトルが常時表示される
- [ ] 90分が8〜10シーンに分割されている

## オンラインライブゆえの制約と対策

| 制約 | 対策 |
|------|------|
| カメラオフが多い | 投票・チャット・リアクションで反応の代替手段を確保 |
| 集中が切れやすい | 10分単位でモード切替、進捗バー常時表示 |
| 沈黙の共有が気まずい | チャット一斉投稿（同時に書く）で沈黙を埋める |
| ブレイクアウト移動の摩擦 | ペアワークは隣同士固定（毎回同じペア）、3人組までに留める |
| 講師1人で全員見られない | チーム発表は代表者制、全員発表は第8回のみに集中 |

## 適用順序の推奨

1. 本ドキュメントを各回設計時の参照原則として固定
2. 第1回の設計書（`design/session-01-design.md`）と教材を団体版へ改訂
3. 第1回を1社で実施 → 反応を見て原則を調整
4. 第2〜8回を順次改訂
5. 業種別ペルソナの設定書（`design/personas.md`）を別途作成
